# 配伍禁忌数据扩展指南

## 概述

本指南旨在帮助开发者和中医专家了解如何扩展和维护中医配伍禁忌数据库，确保系统能够准确识别和管理药物配伍风险。

## 当前数据状况

### 已实现的配伍禁忌数据

1. **十八反**（18组配伍禁忌）
   - 甘草反：甘遂、大戟、海藻、芫花
   - 乌头反：贝母、瓜蒌、半夏、白蔹、白及
   - 藜芦反：人参、沙参、丹参、玄参、细辛、芍药

2. **十九畏**（10组配伍禁忌）
   - 硫黄畏朴硝
   - 水银畏砒霜
   - 狼毒畏密陀僧
   - 巴豆畏牵牛
   - 丁香畏郁金
   - 牙硝畏三棱
   - 川乌、草乌畏犀角
   - 人参畏五灵脂
   - 官桂畏石脂

### 数据结构说明

系统采用以下实体结构存储配伍禁忌数据：

- **Herb**：药材基础信息
- **HerbContraindication**：配伍禁忌关系
- **PrescriptionRisk**：处方风险记录
- **ContraindicationCheckDomain**：配伍禁忌检查领域服务

### 1. 扩展药材库

#### 1.1 常用中药材补充

建议优先补充以下常用中药材：

**补益类**：
- 黄芪、党参、白术、茯苓、山药、熟地黄、当归、白芍、川芎、阿胶

**清热类**：

**解表类**：
- 麻黄、桂枝、紫苏叶、防风、荆芥、薄荷、柴胡、升麻

**理气类**：
- 陈皮、枳壳、木香、香附、青皮、佛手、香橼

**活血类**：
- 红花、桃仁、牛膝、延胡索、三七、蒲黄、五灵脂

```sql
INSERT INTO Herbs (HerbName, Properties, Meridians, Efficacy, Category, CreatedAt, UpdatedAt)

```

### 2. 扩展配伍禁忌数据

#### 2.1 现代研究配伍禁忌

基于现代药理学研究，可以添加以下类型的配伍禁忌：

1. **药物相互作用**
   - 影响吸收的配伍
   - 影响代谢的配伍
   - 影响排泄的配伍

2. **化学配伍禁忌**
   - 产生沉淀的配伍
   - 产生有毒物质的配伍
   - 影响稳定性的配伍

3. **药效学配伍禁忌**
   - 作用相反的配伍
   - 毒性叠加的配伍

#### 2.2 配伍禁忌数据录入模板

```sql
INSERT INTO HerbContraindications 
(PrimaryHerbId, ConflictHerbId, ContraindicationType, RiskLevel, Description, AdverseReactions, Evidence, IsAbsoluteContraindication, IsActive, CreatedAt, UpdatedAt)
VALUES 
(
    (SELECT HerbId FROM Herbs WHERE HerbName = '主药名称'),
    (SELECT HerbId FROM Herbs WHERE HerbName = '配伍药名称'),
    '禁忌类型', -- 十八反/十九畏/现代研究/其他
    风险等级, -- 1-4
    '禁忌描述',
    '可能的不良反应',
    '依据来源',
    是否绝对禁忌, -- 1/0
    1, -- 是否启用
    GETDATE(),
    GETDATE()
);
```

### 3. 数据来源建议

#### 3.1 权威文献来源

1. **古典文献**
   - 《神农本草经》
   - 《本草纲目》
   - 《中华本草》
   - 《中药学》教材

2. **现代研究**
   - 中国药典
   - 药理学研究论文
   - 临床不良反应报告
   - 药物相互作用数据库

#### 3.2 数据质量控制

1. **数据验证**
   - 多源验证：至少两个权威来源支持
   - 专家审核：中医药专家审核确认
   - 临床验证：结合临床实际使用经验

2. **数据分级**
   - A级：权威文献明确记载，临床证据充分
   - B级：文献记载，但临床证据有限
   - C级：理论推测，需要进一步验证

### 4. 特殊情况处理

#### 4.1 相对禁忌的处理

对于一些相对禁忌（非绝对禁忌），可以通过以下方式处理：

1. **设置特殊标记**
   - `IsAbsoluteContraindication = false`
   - 在`SpecialNotes`中说明特殊使用条件

2. **条件性使用**
   - 特定剂量下可以使用
   - 特定炮制方法下可以使用
   - 特定配伍比例下可以使用

#### 4.2 历史争议的处理

对于历史上有争议的配伍禁忌：

1. **保留传统观点**
   - 以传统理论为主
   - 在备注中说明现代观点

2. **风险分级处理**
   - 争议较大的设为中低风险
   - 明确有害的设为高风险

### 5. 数据维护流程

#### 5.1 定期更新机制

1. **季度审核**
   - 检查新发表的相关研究
   - 收集临床反馈意见
   - 更新风险等级评估

2. **年度大修**
   - 全面审核所有配伍禁忌数据
   - 更新药材基础信息
   - 优化数据结构

#### 5.2 反馈收集机制

1. **临床医生反馈**
   - 提供反馈接口
   - 收集实际使用中的问题
   - 记录特殊病例

2. **专家委员会审核**
   - 定期召开专家会议
   - 讨论争议性配伍
   - 制定更新标准
