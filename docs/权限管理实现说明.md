# 权限管理系统实现说明

## 概述

本文档说明基于RBAC（基于角色的访问控制）模型实现的权限管理系统，遵循最小权限原则、租户隔离原则和可追溯原则。

## 核心实体

### 1. Permission（权限表）
**文件位置**: `TcmAiDiagnosis.Entities/Permission.cs`

定义系统中所有可分配的权限点，包括：
- 权限代码（唯一标识）
- 权限名称和描述
- 权限分类和模块
- 是否为系统权限
- 启用状态

### 2. Role（角色表）
**文件位置**: `TcmAiDiagnosis.Entities/Role.cs`

定义四级角色体系：
- **平台超级管理员** (PlatformSuperAdmin)
- **租户管理员** (TenantAdmin)
- **中医师** (Doctor)
- **患者** (Patient)

增强字段：
- 角色类型枚举
- 租户关联（支持租户级角色）
- 系统角色标识
- 审计字段

### 3. RolePermission（角色权限关联表）
**文件位置**: `TcmAiDiagnosis.Entities/RolePermission.cs`

实现角色与权限的多对多关联：
- 支持权限授予和拒绝
- 记录分配人和分配时间
- 支持权限覆盖场景

### 4. PermissionChangeLog（权限变更日志表）
**文件位置**: `TcmAiDiagnosis.Entities/PermissionChangeLog.cs`

记录所有权限分配、变更操作：
- 变更类型（授予、撤销、修改）
- 目标信息（角色或用户）
- 变更前后状态
- 申请人、审批人信息
- 审批流程状态
- 客户端IP和用户代理
- 支持跨租户操作记录

### 5. TemporaryPermission（临时权限表）
**文件位置**: `TcmAiDiagnosis.Entities/TemporaryPermission.cs`

支持特殊场景的临时权限授予：
- **会诊场景**: 临时查看患者病历
- **跨租户转诊**: 临时访问转诊患者信息
- **系统维护**: 临时访问租户数据

特性：
- 有效期控制
- 患者授权机制
- 管理员批准流程
- 访问次数限制
- 自动回收机制

### 6. Tenant（租户表）- 增强版
**文件位置**: `TcmAiDiagnosis.Entities/Tenant.cs`

增强字段：
- 租户代码（唯一标识）
- Logo URL
- 服务套餐类型
- 用户数量限制
- 存储空间配额
- 时区和语言设置
- 软删除支持
- 完整审计字段

## 权限常量定义

**文件位置**: `TcmAiDiagnosis.Domain/Constants/PermissionConstants.cs`

定义所有系统权限点，按角色和功能模块组织：

### 平台超级管理员权限
- 租户管理（创建、编辑、删除、启用/禁用等）
- 系统配置（全局参数、知识库、监控规则）
- 权限管控（定义角色模板、查看所有权限）
- 运维监控（资源监控、日志查看、技术支持）
- 数据安全（加密配置、备份恢复、隔离状态）

### 租户管理员权限
- 用户管理（创建医生、分配角色、重置密码）
- 租户配置（基础信息、功能模块、业务流程）
- 监控管理（使用统计、操作日志、问题处理）
- 数据管理（存储使用、备份策略、数据异常）

### 中医师权限
- 患者管理（创建、编辑、查询、关联、跨租户添加）
- 诊疗操作（就诊记录、四诊采集、AI诊断、确认诊断）
- 治疗方案（创建、编辑、AI生成、版本管理、风险处理）
- 病历管理（创建、编辑、签名、打印、导出、会诊共享）
- 知识库（查询理论、方剂、中药、穴位、诊疗规则）
- 医案管理（查看、审核、编辑、确认、导出）

### 患者权限
- 个人信息（查看基本信息、医疗信息）
- 诊疗记录（查看就诊历史、四诊信息、诊断结果）
- 治疗方案（查看处方、非药物疗法、个性化建议）
- 病历查看（查看、打印、导出）

### 特殊场景权限
- 会诊场景（查看共享病历、参与会诊）
- 跨租户转诊（传递病历、接收病历、查看转诊记录）
- 系统维护（访问租户数据、执行维护）

## 权限验证层级

### 1. 页面级验证
根据用户角色权限动态展示菜单和操作按钮，无权限功能不可见。

### 2. 接口级验证
所有API接口进行权限校验：
- 无权限访问返回403错误
- 记录异常访问日志

### 3. 数据级验证
通过租户ID和用户ID进行数据过滤：
- 确保租户数据隔离
- 用户仅可访问关联数据

## 权限分配流程

### 1. 平台超级管理员
- 由平台运营团队指定
- 线下审批后系统初始化分配
- 账号启用需双人验证

### 2. 租户管理员
- 创建租户时同步创建
- 初始权限由平台角色模板自动分配
- 可由平台超级管理员调整

### 3. 中医师
- 由租户管理员创建并分配角色
- 自动继承角色基础权限
- 可根据专业方向微调权限

### 4. 患者
- 由中医师创建或自行注册
- 注册后自动分配患者角色权限

## 权限变更流程

### 1. 申请
提交权限变更申请，说明：
- 变更原因
- 变更内容
- 有效期

### 2. 审批
根据权限级别进行审批：
- 平台超级管理员权限变更 → 平台运营负责人审批
- 租户内角色权限变更 → 租户管理员审批
- 跨租户权限变更 → 双方租户管理员 + 平台超级管理员审批

### 3. 执行
审批通过后，由权限管理员执行变更操作

### 4. 记录
系统自动记录全流程信息，生成审计日志（保留期≥3年）

## 特殊场景处理

### 1. 会诊场景
- 参与会诊的医生临时获得患者病历查看权限
- 需要患者授权
- 会诊结束后权限自动回收
- 记录授权日志

### 2. 跨租户转诊场景
- 原租户医生可传递脱敏病历给转诊租户医生
- 接收方仅可查看转诊相关病历
- 不可修改病历
- 转诊完成后权限自动终止

### 3. 系统维护场景
- 平台超级管理员访问租户数据需提前报备
- 需获得租户管理员批准
- 访问过程全程记录
- 维护结束后立即回收临时权限

## 审计要求

### 1. 审计日志内容
- 操作人、操作时间、操作类型
- 变更前后权限
- 客户端IP
- 操作结果

### 2. 日志查看权限
- 平台超级管理员：查看全平台日志
- 租户管理员：查看本租户日志
- 日志不可篡改、删除

### 3. 定期审计
- 平台运营团队：每季度审计全平台权限
- 租户管理员：每月审计本租户权限
- 及时清理冗余权限

## 数据库表关系

```
User (用户表)
  ├─ 1:N → Role (角色表)
  │         ├─ N:M → Permission (权限表) [通过 RolePermission]
  │         └─ N:1 → Tenant (租户表)
  ├─ 1:N → PermissionChangeLog (权限变更日志)
  ├─ 1:N → TemporaryPermission (临时权限)
  └─ N:1 → Tenant (租户表)

Permission (权限表)
  ├─ N:M → Role (角色表) [通过 RolePermission]
  ├─ 1:N → PermissionChangeLog (权限变更日志)
  └─ 1:N → TemporaryPermission (临时权限)

Tenant (租户表)
  ├─ 1:N → User (用户表)
  ├─ 1:N → Role (角色表)
  ├─ 1:N → PermissionChangeLog (权限变更日志)
  └─ 1:N → TemporaryPermission (临时权限)
```

## 注意事项

1. **安全性**
   - 所有权限操作必须记录审计日志
   - 敏感操作需要二次验证
   - 定期审计权限分配情况

2. **性能**
   - 实现权限缓存机制
   - 避免频繁查询数据库
   - 使用Redis缓存用户权限

3. **可维护性**
   - 权限代码使用常量定义
   - 权限分类清晰
   - 文档及时更新

4. **扩展性**
   - 预留自定义权限扩展点
   - 支持动态权限配置
   - 支持权限继承和覆盖
