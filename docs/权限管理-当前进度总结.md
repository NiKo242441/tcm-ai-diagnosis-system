## 验证步骤

### 1. 检查权限表数据
```sql
-- 连接数据库
USE `tcm-ai-diagnosis`;

-- 检查权限数量
SELECT COUNT(*) as permission_count FROM permissions;
-- 预期结果：约110个权限

-- 查看权限分类统计
SELECT Category, COUNT(*) as count 
FROM permissions 
GROUP BY Category 
ORDER BY count DESC;

-- 查看所有权限
SELECT PermissionId, PermissionCode, PermissionName, Category, Module 
FROM permissions 
ORDER BY Category, Module, SortOrder;
```

### 2. 检查角色权限关联
```sql
-- 检查角色权限关联数量
SELECT COUNT(*) as role_permission_count FROM role_permissions;
-- 预期结果：约92个关联（24+51+11+6）

-- 查看各角色的权限数量
SELECT r.Name, r.ShowName, COUNT(rp.PermissionId) as permission_count
FROM AspNetRoles r
LEFT JOIN role_permissions rp ON r.Id = rp.RoleId
GROUP BY r.Id, r.Name, r.ShowName
ORDER BY permission_count DESC;

-- 查看Manager角色的权限
SELECT p.PermissionCode, p.PermissionName, p.Category
FROM role_permissions rp
JOIN permissions p ON rp.PermissionId = p.PermissionId
JOIN AspNetRoles r ON rp.RoleId = r.Id
WHERE r.Name = 'Manager'
ORDER BY p.Category, p.SortOrder;
```

### 3. 如果数据未初始化
如果上述查询返回0条记录，说明数据初始化被跳过（可能是因为之前已有数据）。

**解决方案A：清空表并重启应用**
```sql
-- 清空权限相关表
DELETE FROM role_permissions;
DELETE FROM permissions;

-- 重启应用程序，数据初始化会自动运行
```

**解决方案B：手动运行初始化**
在应用程序中添加一个管理页面或API端点来手动触发数据初始化。

## 技术要点

### 权限代码命名规范
- 格式：`{模块}.{操作}`
- 示例：`patient.create`, `diagnosis.use_ai_diagnosis`
- 特殊场景：`consultation.participate`, `referral.transfer_record`

### 数据库表结构
- `permissions` - 权限定义表（110+条记录）
- `role_permissions` - 角色权限关联表（92条记录）
- `permission_change_logs` - 权限变更日志表
- `temporary_permissions` - 临时权限表

### 关键设计决策
1. 使用RBAC模型，支持角色继承
2. 权限代码使用字符串常量，便于维护
3. 支持临时权限，满足特殊场景需求
4. 完整的审计日志，记录所有权限变更
5. 多租户隔离，租户级角色与平台级角色分离
