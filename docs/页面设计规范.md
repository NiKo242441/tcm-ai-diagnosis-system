# 中医数字化诊疗平台页面设计规范

## 1. 概述

本文档定义了中医数字化诊疗平台的页面设计规范，基于PlainAdmin主题和Create.cshtml页面的设计风格制定，旨在确保整个平台的视觉一致性和用户体验的统一性。

## 2. 设计理念

### 2.1 核心原则
- **现代化设计**：采用PlainAdmin的简洁现代风格
- **专业性**：体现医疗系统的专业性和权威性
- **易用性**：简洁明了的界面，降低用户学习成本
- **一致性**：统一的视觉语言和交互模式
- **可访问性**：支持不同设备和用户群体的访问需求

### 2.2 设计风格
- **PlainAdmin风格**：采用现代化的卡片式设计和优雅的阴影效果
- **医疗专业感**：保持医疗系统的专业性和可信度
- **信息层次**：清晰的信息架构和视觉层次
- **响应式设计**：适配各种设备和屏幕尺寸

## 14. Razor页面开发规范

### 14.1 Area页面Layout规范
- **禁止指定Layout**：在Area中的Razor页面（如Doctor、Manager、User等区域），请勿在页面顶部指定Layout
- **自动继承**：Area页面会自动继承全局Layout设置，无需手动指定
- **示例**：
  ```csharp
  @page
  @model TcmAiDiagnosis.Web.Areas.Doctor.Pages.Patient.CreateModel
  // 不要添加 @{ Layout = "_Layout"; }
  ```

### 14.2 页面头部规范
- **统一头部结构**：所有Area页面必须包含标准化的头部结构
- **必需元素**：页面标题、面包屑导航、图标
- **标准模板**：
  ```html
  <div class="title-wrapper pt-30">
      <div class="row align-items-center">
          <div class="col-md-8 row">
              <div class="title col">
                  <h2><i class="fas fa-[图标名称] me-2"></i>[页面标题]</h2>
              </div>
          </div>
          <div class="col-md-4">
              <div class="breadcrumb-wrapper">
                  <nav aria-label="breadcrumb">
                      <ol class="breadcrumb">
                          <li class="breadcrumb-item">
                              <a href="[父级页面链接]"><i class="fas fa-[父级图标] me-1"></i>[父级名称]</a>
                          </li>
                          <li class="breadcrumb-item active" aria-current="page">
                              [当前页面名称]
                          </li>
                      </ol>
                  </nav>
              </div>
          </div>
      </div>
  </div>
  ```

### 14.3 头部样式规范
- **容器类**：使用 `title-wrapper pt-30` 作为头部容器
- **标题样式**：使用 `<h2>` 标签，配合FontAwesome图标
- **图标间距**：图标与文字间使用 `me-2` 类设置间距
- **面包屑导航**：使用Bootstrap的 `breadcrumb` 组件
- **响应式布局**：使用 `col-md-8` 和 `col-md-4` 实现响应式布局

### 14.4 外部文件引用规范
- **统一管理**：所有外部CSS、JavaScript文件必须在 `_Layout.cshtml` 中统一引用
- **禁止页面级引用**：不允许在单个页面中直接引用外部文件
- **Layout文件位置**：`TcmAiDiagnosis.Web\Pages\Shared\_Layout.cshtml`
- **引用示例**：
  ```html
  <!-- CSS文件引用 -->
  <link rel="stylesheet" href="~/assets/css/bootstrap.min.css" />
  <link rel="stylesheet" href="~/css/medical-system-styles.css" asp-append-version="true" />
  
  <!-- JavaScript文件引用 -->
  <script src="~/assets/js/bootstrap.bundle.min.js"></script>
  <script src="~/js/site.js" asp-append-version="true"></script>
  ```

### 14.5 公共样式管理规范
- **公共样式文件**：`TcmAiDiagnosis.Web\Pages\Shared\_Layout.cshtml.css`
- **样式作用域**：仅用于全局通用样式，避免页面特定样式
- **命名规范**：遵循医疗系统样式命名规范（`medical-` 前缀）
- **样式示例**：
  ```css

      color: white;
  }

  }
  ```

### 14.6 超链接和路由规范
- **路由确认**：创建超链接前必须查看目标页面的 `@page` 指令
- **参数传递**：根据页面路由定义确认参数传递方式
- **检查步骤**：
  1. 查看目标页面顶部的 `@page` 指令
  2. 确认路由模板和参数要求
  3. 按照路由模板构建正确的链接
- **示例分析**：
  ```csharp
  // 目标页面路由定义
  @page "/Doctor/Patient/Details/{id:int}"
  
  // 正确的链接构建
  <a href="/Doctor/Patient/Details/@patient.Id">查看详情</a>
  
  // 或使用Razor语法
  <a asp-area="Doctor" asp-page="/Patient/Details" asp-route-id="@patient.Id">查看详情</a>
  ```

### 14.7 页面参数传递规范
- **路由参数**：通过URL路径传递的参数（如 `{id:int}`）
- **查询参数**：通过查询字符串传递的参数（如 `?name=value`）
- **表单参数**：通过POST请求传递的参数
- **参数验证**：在PageModel中进行参数验证和类型转换
- **示例**：
  ```csharp
  // 页面路由定义
  @page "/Doctor/Patient/Edit/{id:int?}"
  
  // PageModel中的参数处理
  public class EditModel : PageModel
  {
      [BindProperty(SupportsGet = true)]
      public int? Id { get; set; }
      
      public IActionResult OnGet()
      {
          if (!Id.HasValue)
          {
              return RedirectToPage("/Patient/Index");
          }
          // 处理逻辑
          return Page();
      }
  }
  ```

### 14.8 Area页面命名规范
- **文件命名**：使用PascalCase命名法（如 `Create.cshtml`、`Details.cshtml`）
- **页面标题**：与文件名保持一致的语义
- **路由命名**：遵循RESTful风格的命名约定
- **示例结构**：
  ```
  Areas/
  ├── Doctor/
  │   └── Pages/
  │       └── Patient/
  │           ├── Index.cshtml      (患者列表)
  │           ├── Create.cshtml     (新建患者)
  │           ├── Details.cshtml    (患者详情)
  │           ├── Edit.cshtml       (编辑患者)
  │           └── Delete.cshtml     (删除患者)
  ```

### 14.9 页面模型绑定规范
- **属性绑定**：使用 `[BindProperty]` 特性进行模型绑定
- **GET参数**：使用 `[BindProperty(SupportsGet = true)]` 支持GET请求参数
- **验证特性**：使用数据注解进行模型验证
- **示例**：
  ```csharp
  public class CreateModel : PageModel
  {
      [BindProperty]
      [Required(ErrorMessage = "患者姓名不能为空")]
      [StringLength(50, ErrorMessage = "患者姓名长度不能超过50个字符")]
      public string PatientName { get; set; }
      
      [BindProperty]
      [Range(0, 150, ErrorMessage = "年龄必须在0-150之间")]
      public int Age { get; set; }
  }
  ```
