# 开发计划3：治疗方案内容管理与编辑（医生修改调整阶段）

## 1. 项目定位与功能概述

治疗方案内容管理与编辑功能是医生在AI生成初步治疗方案后，进行个性化调整和优化的核心模块。该模块承担着将AI智能分析与医生专业判断相结合的重要职责，确保最终治疗方案既符合中医理论体系，又满足患者个体化需求。

### 1.1 功能边界声明（系统级标准 v4.0）

**本模块严格遵循**以下边界原则：
- ✅ **仅处理AIGenerated(1)状态的治疗方案**：仅接收计划2生成的AIGenerated状态治疗方案
- ✅ **提供个性化编辑功能**：允许医生根据患者特殊情况调整治疗方案
- ✅ **支持多维度编辑**：涵盖中药、针灸、艾灸、拔罐、推拿、食疗六大体系完整数据结构
- ✅ **状态流转管理**：编辑完成后自动触发计划4的禁忌检查流程
- ✅ **版本规范**：编辑完成后必须生成新版本（由计划5处理），版本号从V1.0.0开始递增
- ❌ **不重新生成完整方案**：不提供重新生成完整治疗方案的功能
- ❌ **不修改AI原始数据**：AI生成的原始数据作为历史版本保留，仅创建新版本
- ❌ **不处理非AI生成方案**：仅处理IsAiOriginated=true的治疗方案

### 1.2 状态流转机制（系统级标准 v4.0）

#### 1.2.1 编辑触发条件（强制规范）
- **状态限制**：仅允许处理`TreatmentStatus.AIGenerated`或`TreatmentStatus.CheckFailed`状态的治疗方案
- **来源限制**：仅允许编辑AI生成的治疗方案（IsAiOriginated必须为true）
- **权限验证**：医生必须具有该患者的治疗方案编辑权限，基于CreatedByUserId统一字段验证
- **版本要求**：必须基于最新版本（IsLatest = true）进行编辑，版本号为V1.0.0

#### 1.2.2 标准状态流转流程（与计划1完全统一）
```
计划2.AIGenerated → 计划3.Editing → 计划4.Checking → 
计划4.CheckFailed → 计划3.Editing → 计划4.Checking → 
计划4.Checking → 计划5.Versioning → 计划6.Finalized
```

#### 1.2.3 状态流转详细规范
| 当前状态 | 触发条件 | 下一状态 | 负责模块 | 备注 |
|----------|----------|----------|----------|------|
| AIGenerated | 医生点击"开始编辑" | Editing | 计划3 | 创建编辑会话 |
| Editing | 医生保存编辑内容 | Checking | 计划4 | 自动触发禁忌检查 |
| CheckFailed | 医生修改后重新提交 | Checking | 计划4 | 重新进行禁忌检查 |
| Checking | 检查通过 | Versioning | 计划5 | 自动生成新版本 |
| Versioning | 版本生成完成 | Finalized | 计划7 | 可打印导出 |

#### 1.2.4 编辑完成后的标准处理
- **自动触发**：编辑完成后自动调用计划4的配伍禁忌检查接口
- **状态验证**：编辑完成后必须将状态从Editing更新为Checking
- **版本控制**：检查通过后自动进入计划5的版本管理流程
- **审计记录**：所有编辑操作必须记录到计划6的审计日志中

#### 1.2.5 边界声明（与计划2/4/5/6的接口）
- **接收数据**：仅接收计划2生成的AIGenerated状态治疗方案
- **输出数据**：仅向计划4输出Checking状态的治疗方案
- **版本管理**：不处理版本号生成，由计划5统一处理
- **审计日志**：所有编辑操作通过计划6统一记录

### 1.3 核心定位
- **AI与医生协作的桥梁**：将AI算法生成的标准化方案转化为个性化治疗方案
- **专业决策支持系统**：为医生提供全面的编辑工具和决策支持
- **医疗安全保障**：集成多层次风险管控机制，确保用药安全
- **版本化管理体系**：实现治疗方案的完整生命周期管理

## 2. 功能需求（治疗方案内容编辑核心）

### 2.1 治疗方案基础信息管理

#### 2.1.1 诊断信息维护

### 2.2 中药处方编辑管理

#### 2.2.1 处方结构管理
**处方基本信息**
- 方剂名称编辑（支持自定义命名）
- 处方类型选择（汤剂、丸剂、散剂等）
- 处方功效描述（主治、适应症等）
- 处方用法用量（服用方法、频次、疗程）

**多处方管理**
- 一个治疗方案可包含多个中药处方
- 处方间顺序调整（拖拽排序）
- 处方启用/禁用状态切换
- 处方复制功能（快速创建相似处方）

#### 2.2.2 药材明细管理
**药材操作**
- 药材添加：支持药材库搜索、拼音首字母检索
- 药材删除：支持单条删除和批量删除
- 药材修改：用量、炮制方法、特殊用法调整
- 药材替换：一键替换为功效相近药材

**药材属性编辑**
- 用量设置：支持g、钱、两等多种单位
- 炮制方法：炒、炙、煅、蒸等20+种选项
- 特殊用法：先煎、后下、包煎、烊化等标记
- 药材备注：添加个性化用药说明

**智能辅助编辑**
- 药材功效提示：实时显示药材主要功效
- 用量范围提醒：显示安全用量区间
- 相互作用提示：药材间可能的影响关系
- 常用组合推荐：基于处方的智能推荐

### 2.3 针灸治疗方案编辑

```csharp
public class EditRiskValidator : IEditRiskValidator
{
    public async Task<EditRiskResult> ValidateEditRiskAsync(EditRiskRequest request)
    {
        var risks = new List<EditRisk>();
        var score = 0;

        // 毒性药材检查
        score += await CheckToxicHerbsRisk(request);
        
        // 特殊人群检查
        score += await CheckSpecialPopulationRisk(request);
        
        // 安全警告检查
        score += await CheckSafetyWarningRisk(request);
        
        // 系统协同检查
        score += await CheckSystemInteractionRisk(request);

        return new EditRiskResult
        {
            RequiresVerification = score >= _riskThreshold,
            RiskScore = score,
            Risks = risks,
            Recommendations = GenerateRecommendations(risks)
        };
    }

    private async Task<int> CheckToxicHerbsRisk(EditRiskRequest request)
    {
        var toxicHerbs = new[] { "附子", "乌头", "马钱子", "巴豆" };
        var modifiedHerbs = request.ModifiedData.HerbalPrescriptions
            .SelectMany(p => p.Items)
            .Where(item => toxicHerbs.Contains(item.Name));

        if (modifiedHerbs.Any())
        {
            return 3; // 高风险
        }
        return 0;
    }
}
```

```csharp
        var riskFactors = new List<RiskFactor>();
        var riskScore = 0;

        // 1. 毒性药材变更检查
        riskScore += await AssessToxicHerbChanges(request);
        
        // 2. 配伍禁忌新增检查
        riskScore += await AssessContraindicationChanges(request);
        
        // 3. 特殊人群用药检查
        riskScore += await AssessSpecialPopulationChanges(request);
        
        // 4. 治疗方案完整性检查
        riskScore += await AssessTreatmentIntegrityChanges(request);

        return new EditRiskResult
        {
            RiskLevel = DetermineRiskLevel(riskScore),
            RiskFactors = riskFactors,
            RequiresVerification = riskScore >= 5,
            VerificationMethod = DetermineVerificationMethod(riskScore)
        };
    }

    private RiskLevel DetermineRiskLevel(int score)
    {
        return score switch
        {
            >= 10 => RiskLevel.Critical,
            >= 7 => RiskLevel.High,
            >= 5 => RiskLevel.Medium,
            _ => RiskLevel.Low
        };
    }
}
```

#### 8.5.3 编辑验证服务实现

**分层验证服务**：
```csharp
public class EnhancedEditVerificationService : IEnhancedEditVerificationService
{
    public async Task<EditVerificationResult> ProcessEditVerificationAsync(
        EditVerificationRequest request)
    {
        var assessment = await _riskAssessmentService.AssessEditRiskAsync(request);
        
        switch (assessment.RiskLevel)
        {
            case RiskLevel.Low:
                return new EditVerificationResult { Approved = true };
                
            case RiskLevel.Medium:
                return await ProcessMediumRiskVerification(request);
                
            case RiskLevel.High:
                return await ProcessHighRiskVerification(request);
                
            case RiskLevel.Critical:
                return await ProcessCriticalRiskVerification(request);
                
            default:
                throw new ArgumentOutOfRangeException();
        }
    }

    private async Task<EditVerificationResult> ProcessHighRiskVerification(
        EditVerificationRequest request)
    {
        // 1. 验证医生密码
        var passwordValid = await _userService.ValidateDoctorPasswordAsync(
            request.DoctorId, request.Password);
        
        if (!passwordValid)
            return EditVerificationResult.Failed("医生密码验证失败");

        // 2. 创建高风险验证记录
        var record = new DoctorVerificationRecord
        {
            DoctorId = request.DoctorId,
            TreatmentId = request.TreatmentId,
            RiskLevel = RiskLevel.High,
            VerificationMethod = VerificationMethod.Password,
            VerificationStatus = VerificationStatus.Approved,
            VerificationReason = request.ClinicalReason,
            CreatedAt = DateTime.UtcNow,
            IpAddress = request.IpAddress,
            UserAgent = request.UserAgent
        };

        await _verificationRepository.SaveAsync(record);
        await _auditLogService.LogEditVerificationAsync(record);

        return EditVerificationResult.Success(record.Id);
    }

    private async Task<EditVerificationResult> ProcessCriticalRiskVerification(
        EditVerificationRequest request)
    {
        // 1. 验证医生密码
        var passwordValid = await _userService.ValidateDoctorPasswordAsync(
            request.DoctorId, request.Password);
        
        if (!passwordValid)
            return EditVerificationResult.Failed("医生密码验证失败");

        // 2. 创建极高风险验证记录（待审核状态）
        var record = new DoctorVerificationRecord
        {
            DoctorId = request.DoctorId,
            TreatmentId = request.TreatmentId,
            RiskLevel = RiskLevel.Critical,
            VerificationMethod = VerificationMethod.ClinicReview,
            VerificationStatus = VerificationStatus.PendingReview, // 待审核状态
            VerificationReason = request.ClinicalJustification,
            ReviewStatus = ReviewStatus.Pending, // 新增：初始为待审核
            CreatedAt = DateTime.UtcNow,
            IpAddress = request.IpAddress,
            UserAgent = request.UserAgent
        };

        await _verificationRepository.SaveAsync(record);
        
        // 3. 发送实时审核通知给诊所药剂师/管理员
        await _notificationService.NotifyReviewersAsync(new ReviewNotification
        {
            TreatmentId = request.TreatmentId,
            DoctorId = request.DoctorId,
            RiskLevel = RiskLevel.Critical,
            VerificationRecordId = record.Id,
            ClinicalReason = request.ClinicalJustification,
            CreatedAt = DateTime.UtcNow
        });

        await _auditLogService.LogCriticalEditVerificationAsync(record);

        return EditVerificationResult.PendingReview(record.Id, "已提交诊所药剂师/管理员审核");
    }
}
```
