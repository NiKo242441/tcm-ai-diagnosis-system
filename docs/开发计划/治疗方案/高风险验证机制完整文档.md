# 中医数字化诊疗平台 - 高风险操作验证机制完整文档

## 1. 机制概述与更新背景

### 1.1 机制定位
高风险操作验证机制是贯穿整个治疗方案生命周期的安全保障系统，基于风险分级管理原则，为不同风险等级的操作提供相应的验证流程，确保医疗安全的同时提高操作效率。

### 1.2 更新背景
基于开发计划1-5及原有验证机制总结，本次对《开发计划-治疗方案模块.md》和《需求分析说明书.md》进行了系统性完善，重点补充了风险提示和医生跳过确认相关的内容，将原有的简单风险提示升级为完整的四级风险验证体系。

### 1.3 核心设计原则
- **风险分级**：0-4分低风险、5-6分中风险、7-9分高风险、10+分极高风险
- **验证分层**：确认对话框 → 密码验证 → 多级验证（主任审批）
- **中医特色**：融入经典理论、辨证思维、师承经验
- **全程追踪**：从AI生成到最终使用的完整审计链
- **灵活控制**：明确可跳过与不可跳过场景，平衡安全与效率

### 1.4 覆盖范围
- **开发计划2**：AI治疗方案生成阶段的高风险场景
- **开发计划3**：治疗方案编辑修改阶段的高风险操作
- **开发计划4**：配伍禁忌检查阶段的禁忌确认操作
- **开发计划6**：治疗方案审计与日志管理

## 2. 风险分级标准体系

### 2.1 通用风险评分体系
```csharp
public enum RiskLevel
{
    Low = 0,      // 0-4分：低风险，直接操作
    Medium = 1,   // 5-6分：中风险，确认对话框
    High = 2,     // 7-9分：高风险，密码验证
    Critical = 3  // 10+分：极高风险，多级验证
}
```

### 2.2 验证级别规范表
| 验证级别 | 风险评分 | 触发场景 | 验证方式 | 跳过权限 | 处理特点 |
|----------|----------|----------|----------|----------|----------|
| 一级 | 0-4分 | 标准治疗方案 | 自动通过 | 无需确认 | 系统记录 |
| 二级 | 5-6分 | 毒性药材使用 | 确认对话框 | 可取消操作 | 医生选择 |
| 三级 | 7-9分 | 特殊人群用药、配伍禁忌 | 密码验证 | 不可跳过 | 强制验证 |
| 四级 | 10+分 | 剧毒药物组合、绝对禁忌 | 多级验证 | 不可跳过 | 主任审批 |

### 2.3 各阶段风险场景映射
| 开发阶段 | 低风险(0-4分) | 中风险(5-6分) | 高风险(7-9分) | 极高风险(10+分) |
|----------|---------------|---------------|---------------|-----------------|
| **AI生成** | 标准治疗方案 | 毒性药材使用 | 特殊人群用药 | 剧毒药物组合 |
| **编辑修改** | 常规剂量调整 | 毒性药材增量 | 结构重大调整 | 删除安全警告 |
| **禁忌检查** | 轻微配伍冲突 | 经典相对禁忌 | 妊娠绝对禁忌 | 毒性超限使用 |

## 3. 四级验证机制详解

### 3.1 验证流程状态机
```
开始 → 风险评估 → 风险等级判断 → 
├─ 低风险 → 自动通过 → 记录日志 → 完成
├─ 中风险 → 确认对话框 → 确认/取消 → 完成
├─ 高风险 → 密码验证 → 验证/失败 → 完成
└─ 极高风险 → 多级验证 → 验证/失败 → 完成
```

### 3.2 各级验证详细设计

#### 3.2.1 一级验证：低风险自动通过
- **触发条件**：风险评分0-4分
- **处理方式**：系统自动记录，无需人工干预
- **审计要求**：记录操作日志，无需验证记录
- **用户体验**：无感知通过，操作流畅

#### 3.2.2 二级验证：中风险确认对话框
- **触发条件**：风险评分5-6分
- **处理方式**：显示风险摘要，医生勾选确认
- **界面要素**：
  - 风险标题和简要说明
  - 确认复选框
  - 取消/继续按钮
- **跳过权限**：医生可选择取消操作或继续确认

#### 3.2.3 三级验证：高风险密码验证
- **触发条件**：风险评分7-9分
- **处理方式**：医生密码验证 + 理由说明
- **界面要素**：
  - 详细风险分析
  - 医生密码输入框
  - 临床使用理由文本域
  - 经典理论依据选择
- **跳过权限**：不可跳过，必须完成验证

#### 3.2.4 四级验证：极高风险多级验证
- **触发条件**：风险评分10+分
- **处理方式**：医生密码 + 诊所药剂师/管理员审核
- **界面要素**：
  - 完整风险报告
  - 详细中医辨证分析
  - 经典文献引用
  - 安全保障措施说明
  - 审核人员选择（药剂师/管理员）
  - 审核意见录入
  - 审核状态实时显示
- **审核流程**：
  1. 主治医生提交极高风险操作申请
  2. 系统自动通知指定药剂师或管理员
  3. 审核人员登录系统进行专业审核
  4. 审核通过后主治医生获得操作权限
  5. 所有审核记录完整保存
- **跳过权限**：不可跳过，必须完成多级验证和审核

## 4. 医生跳过确认机制

### 4.1 可跳过场景（医生有选择权）
- **中风险操作提示**：风险评分5-6分的操作
- **轻微配伍冲突警告**：经典相对禁忌的轻微冲突
- **常规剂量调整建议**：基于经验的合理调整
- **一般性的药物相互作用提醒**：非严重的相互作用

**可跳过流程**：
1. 系统显示风险提示
2. 医生可选择"取消操作"或"继续确认"
3. 选择"继续确认"后按风险级别完成验证
4. 记录完整的决策过程

### 4.2 不可跳过场景（强制验证）
- **极高风险操作**：风险评分≥10分的操作
- **配伍禁忌强制检查失败**：绝对禁忌的配伍组合
- **特殊人群绝对禁忌用药**：妊娠、儿童等特殊人群
- **毒性药材超安全剂量使用**：超出安全范围的毒性药材
- **剧毒药物组合使用**：具有协同毒性的药物组合
- **妊娠绝对禁忌用药**：明确妊娠禁忌的药物

**不可跳过流程**：
1. 系统直接显示对应级别的验证界面
2. 必须完成验证才能继续操作
3. 无取消或跳过选项
4. 验证记录强制保存

public class DoctorVerificationRecord

## 8. 审计与合规要求

### 8.1 完整审计链
每个高风险操作都生成完整的审计记录：
- **操作前**：风险评估报告
- **操作中**：验证过程记录（包括跳过决策）
- **操作后**：执行结果追踪

### 8.2 合规要求
- **数据保留**：所有验证记录保存10年
- **隐私保护**：敏感信息脱敏处理
- **可追溯性**：支持多维度查询和追溯
- **合规标准**：
  - 符合《中医药法》相关要求
  - 满足《电子病历管理规范》
  - 通过医疗信息安全等级保护
