# 开发计划5：治疗方案版本管理

## 1. 功能定位与业务边界

治疗方案版本管理是治疗方案全生命周期管理中的核心环节，位于**治疗方案内容管理与编辑**之后，**治疗方案审计与日志**之前。该功能确保每一次治疗方案的修改都能被完整记录，支持历史版本的追溯、对比和恢复，为治疗方案的持续优化提供数据支撑。

### 1.1 前置业务依赖（系统级标准 v4.0）
- **计划4职责**：仅处理Versioning(5)状态的治疗方案，必须完成配伍禁忌检查且检查通过
- **数据完整性**：必须包含六大治疗体系完整数据结构（中药、针灸、艾灸、拔罐、推拿、食疗）
- **版本规范**：所有版本必须遵循V{主版本}.{次版本}.{修订号}格式，AI生成方案初始版本必须为V1.0.0

### 1.2 后置业务支撑
- **计划6职责**：版本管理产生的操作日志必须记录到计划6的审计日志中
- **计划7职责**：仅向计划7输出Finalized(6)状态的治疗方案，支持基于任意历史版本进行打印和导出

### 1.3 状态流转机制（系统级标准 v4.0）

#### 1.3.1 版本管理触发条件（强制规范）
- **状态限制**：仅处理`TreatmentStatus.Versioning`状态的治疗方案
- **前置条件**：必须完成计划4的配伍禁忌检查且检查通过
- **版本要求**：必须基于最新版本（IsLatest = true）进行版本管理
- **权限验证**：医生必须具有版本管理权限

#### 1.3.2 版本命名规范（系统级强制标准）
- **格式**：必须与计划1统一规范完全匹配：`V{主版本}.{次版本}.{修订号}`
- **版本号递增规则**（与计划1完全统一）：
  - **主版本（Major）**：治疗架构重构（如从中药为主转为针灸为主）
  - **次版本（Minor）**：核心内容调整（如主药/主穴变更）
  - **修订号（Patch）**：细节优化（如剂量微调、描述优化）
- **初始版本**：所有AI生成的治疗方案初始版本必须为`V1.0.0`
- **版本兼容性**：必须与计划1所有实体的Version字段100%兼容

#### 1.3.3 标准状态流转机制（与计划1完全统一）
```csharp
// 版本生成完成后的标准状态更新（强制遵循计划1规范）
if (versionResult.IsSuccess)
{
    // 新版本处理
    newVersion.Status = TreatmentStatus.Finalized;  // 计划1统一枚举值6
    newVersion.IsLatest = true;
    newVersion.Version = newVersionNumber;          // 标准三段式格式
    newVersion.FinalizedAt = DateTime.UtcNow;

    // 旧版本处理
    oldVersion.IsLatest = false;
    oldVersion.Status = TreatmentStatus.Archived;   // 计划1统一枚举值7
    oldVersion.ArchivedAt = DateTime.UtcNow;
    
    // 自动触发计划7的打印导出准备
}
else
{
    // 版本生成失败，保持Versioning状态
    treatment.Status = TreatmentStatus.Versioning;
    treatment.VersionError = versionResult.ErrorMessage;
}
```

#### 1.3.4 状态流转详细规范
| 当前状态 | 触发条件 | 操作结果 | 下一状态 | 负责模块 | 备注 |
|----------|----------|----------|----------|----------|------|
| Versioning | 版本生成成功 | 新版本Finalized | Finalized | 计划7 | 可打印导出 |
| Versioning | 版本生成失败 | 保持Versioning | Versioning | 计划5 | 记录错误日志 |
| Finalized | 后续编辑 | 新版本生成 | Versioning | 计划3 | 重新进入编辑流程 |
| Finalized | 新版本生成 | 旧版本归档 | Archived | 计划5 | 历史版本保留 |

#### 1.3.5 与计划1/3/4/6/7的接口标准
- **接收数据**：仅接收计划4输出的Versioning状态治疗方案
- **输出数据**：仅向计划7输出Finalized状态的治疗方案
- **版本管理**：统一负责所有版本号的生成和管理
- **审计记录**：所有版本操作必须记录到计划6的审计日志中
- **历史保留**：自动将旧版本标记为Archived状态，供历史查询

##### 版本号生成规则详解
基于中医治疗方案的实际变更特点，采用`V{主版本号}.{次版本号}.{修订号}`的三段式版本命名规则，专注于治疗方案内容的实质性变更：

**版本号递增逻辑：**
- **主版本号（Major）**：治疗方案架构重构时递增（很少发生）
  - 治疗手段类型重大调整（如从纯中药转为针灸为主）
  - 治疗周期从急性转为慢性长期调理
  - 整体治疗思路的根本性转变
- **次版本号（Minor）**：治疗方案内容调整时递增（常见）
  - 中药处方：主药组成变化（新增/替换/删除核心药材）
  - 针灸方案：主穴配伍重大调整（新增/替换核心穴位）
  - 治疗频次：从每日调整为隔日或每周等显著变化
  - 食疗方案：主要食疗方剂的替换或新增
- **修订号（Patch）**：治疗方案细节优化时递增（最频繁）
  - 中药剂量：单味药材剂量的微调（±20%以内）
  - 穴位手法：针刺深度、留针时间的细微调整
  - 文字描述：医嘱、注意事项、服用方法的优化
  - 配伍加减：基于患者反馈的轻微化裁调整

```csharp
using Microsoft.EntityFrameworkCore;
using TcmAiDiagnosis.EFContext;

public class TreatmentVersionGenerator
{
    private readonly TcmAiDiagnosisContext _context;
    
    public TreatmentVersionGenerator(TcmAiDiagnosisContext context)
    {
        _context = context;
    }
    
    public async Task<string> GenerateVersionAsync(int treatmentId, VersionStatus changeType)
    {
        var lastVersion = await _context.TreatmentVersions
            .Where(tv => tv.TreatmentId == treatmentId)
            .OrderByDescending(tv => tv.CreatedAt)
            .FirstOrDefaultAsync();
        
        if (lastVersion == null)
        {
            return "V1.0.0"; // 首个版本
        }
        
        var currentVersion = ParseVersion(lastVersion.Version);
        
        return changeType switch
        {
            VersionStatus.Archived => $"V{currentVersion.Major + 1}.0.0",
            VersionStatus.Active => $"V{currentVersion.Major}.{currentVersion.Minor + 1}.0",
            VersionStatus.Draft => $"V{currentVersion.Major}.{currentVersion.Minor}.0",
            _ => throw new ArgumentException("Invalid change type")
        };
    }
    
    private Version ParseVersion(string versionStr)
    {
        var parts = versionStr.TrimStart('V').Split('.');
        return new Version(
            int.Parse(parts[0]), 
            int.Parse(parts[1]), 
            int.Parse(parts[2])
        );
    }
}
```

##### 变更类型自动识别算法

```csharp
public class TreatmentChangeAnalyzer
{
    public VersionStatus AnalyzeChangeType(TreatmentVersion oldVersion, TreatmentVersion newVersion)
    {
        var oldContent = JsonSerializer.Deserialize<TreatmentContent>(oldVersion.ContentSnapshot);
        var newContent = JsonSerializer.Deserialize<TreatmentContent>(newVersion.ContentSnapshot);
        
        // 1. 检查是否为主版本变更（架构重构）
        if (IsMajorChange(oldContent, newContent))
            return VersionStatus.Archived;
            
        // 2. 检查是否为次版本变更（内容调整）
        if (IsMinorChange(oldContent, newContent))
            return VersionStatus.Active;
            
        // 3. 默认为修订号变更（细节优化）
        return VersionStatus.Draft;
    }
    
    private bool IsMajorChange(TreatmentContent old, TreatmentContent newContent)
    {
        // 检查治疗手段类型是否发生根本性变化
        var oldTreatmentTypes = GetTreatmentTypes(old);
        var newTreatmentTypes = GetTreatmentTypes(newContent);
        
        // 如果主要治疗手段发生变化（如从中药为主变为针灸为主）
        if (HasMajorTreatmentTypeChange(oldTreatmentTypes, newTreatmentTypes))
            return true;
            
        // 检查治疗周期是否从急性转为慢性
        if (IsTreatmentCycleMajorChange(old.TreatmentCycle, newContent.TreatmentCycle))
            return true;
            
        return false;
    }
    
    private bool IsMinorChange(TreatmentContent old, TreatmentContent newContent)
    {
        // 中药处方：检查核心药材变化
        if (HasCoreHerbCompositionChange(old.HerbalPrescriptions, newContent.HerbalPrescriptions))
            return true;
            
        // 针灸方案：检查主穴配伍变化
        if (HasCoreAcupunctureChange(old.AcupuncturePoints, newContent.AcupuncturePoints))
            return true;
            
        // 治疗频次：检查显著变化
        if (HasTreatmentFrequencyChange(old.TreatmentFrequency, newContent.TreatmentFrequency))
            return true;
            
        // 食疗方案：检查主要方剂变化
        if (HasCoreDietaryChange(old.DietaryTherapies, newContent.DietaryTherapies))
            return true;
            
        return false;
    }
    
    private List<string> GetTreatmentTypes(TreatmentContent content)
    {
        var types = new List<string>();
        if (content.HerbalPrescriptions?.Any() == true) types.Add("中药");
        if (content.AcupuncturePoints?.Any() == true) types.Add("针灸");
        if (content.CuppingPoints?.Any() == true) types.Add("拔罐");
        if (content.MoxibustionPoints?.Any() == true) types.Add("艾灸");
        return types;
    }
    
    private bool HasMajorTreatmentTypeChange(List<string> oldTypes, List<string> newTypes)
    {
        var oldPrimary = oldTypes.FirstOrDefault();
        var newPrimary = newTypes.FirstOrDefault();
        return oldPrimary != newPrimary;
    }
    
    private bool HasCoreHerbCompositionChange(List<HerbalPrescription> old, List<HerbalPrescription> newPrescriptions)
    {
        if (old?.Count != newPrescriptions?.Count) return true;
        
        for (int i = 0; i < old.Count; i++)
        {
            var oldCoreHerbs = old[i].Herbs?.Where(h => h.IsCore == true).Select(h => h.Name).ToHashSet();
            var newCoreHerbs = newPrescriptions[i].Herbs?.Where(h => h.IsCore == true).Select(h => h.Name).ToHashSet();
            
            if (oldCoreHerbs?.SetEquals(newCoreHerbs) == false) return true;
        }
        return false;
    }
    
    private bool HasCoreAcupunctureChange(List<AcupuncturePoint> old, List<AcupuncturePoint> newPoints)
    {
        var oldCorePoints = old?.Where(p => p.IsCore == true).Select(p => p.PointName).ToHashSet();
        var newCorePoints = newPoints?.Where(p => p.IsCore == true).Select(p => p.PointName).ToHashSet();
        
        return oldCorePoints?.SetEquals(newCorePoints) == false;
    }
}
```

**字段级变更追踪系统**
```csharp
public class FieldLevelChangeTracker
{
    public List<FieldChangeDetail> TrackFieldChanges(TreatmentVersion oldVersion, TreatmentVersion newVersion)
    {
        var changes = new List<FieldChangeDetail>();
        
        // 中药处方字段级对比
        changes.AddRange(CompareHerbalPrescriptions(oldVersion.HerbalPrescriptions, newVersion.HerbalPrescriptions));
        
        // 针灸穴位字段级对比
        changes.AddRange(CompareAcupuncturePoints(oldVersion.AcupuncturePoints, newVersion.AcupuncturePoints));
        
        // 艾灸方案字段级对比
        changes.AddRange(CompareMoxibustionPoints(oldVersion.MoxibustionPoints, newVersion.MoxibustionPoints));
        
        // 生活方式建议字段级对比
        changes.AddRange(CompareLifestyleAdvice(oldVersion.LifestyleAdvice, newVersion.LifestyleAdvice));
        
        return changes;
    }
    
    private List<FieldChangeDetail> CompareHerbalPrescriptions(List<HerbalPrescription> old, List<HerbalPrescription> newPrescriptions)
    {
        var changes = new List<FieldChangeDetail>();
        
        // 检查每个处方的具体字段变更
        for (int i = 0; i < Math.Max(old?.Count ?? 0, newPrescriptions?.Count ?? 0); i++)
        {
            var oldPrescription = i < (old?.Count ?? 0) ? old[i] : null;
            var newPrescription = i < (newPrescriptions?.Count ?? 0) ? newPrescriptions[i] : null;
            
            if (oldPrescription == null && newPrescription != null)
            {
                changes.Add(new FieldChangeDetail { Field = $"处方{i+1}", ChangeType = "新增", OldValue = "", NewValue = newPrescription.Name });
            }
            else if (oldPrescription != null && newPrescription == null)
            {
                changes.Add(new FieldChangeDetail { Field = $"处方{i+1}", ChangeType = "删除", OldValue = oldPrescription.Name, NewValue = "" });
            }
            else if (oldPrescription != null && newPrescription != null)
            {
                // 比较处方名称
                if (oldPrescription.Name != newPrescription.Name)
                    changes.Add(new FieldChangeDetail { Field = $"处方{i+1}名称", ChangeType = "修改", OldValue = oldPrescription.Name, NewValue = newPrescription.Name });
                
                // 比较药材组成
                changes.AddRange(CompareHerbComposition(oldPrescription.Herbs, newPrescription.Herbs, i));
            }
        }
        
        return changes;
    }
    
    private List<FieldChangeDetail> CompareHerbComposition(List<HerbItem> oldHerbs, List<HerbItem> newHerbs, int prescriptionIndex)
    {
        var changes = new List<FieldChangeDetail>();
        
        // 创建药材字典便于比较
        var oldHerbDict = oldHerbs?.ToDictionary(h => h.Name) ?? new Dictionary<string, HerbItem>();
        var newHerbDict = newHerbs?.ToDictionary(h => h.Name) ?? new Dictionary<string, HerbItem>();
        
        // 检查新增药材
        foreach (var newHerb in newHerbs ?? new List<HerbItem>()) 
        {
            if (!oldHerbDict.ContainsKey(newHerb.Name))
            {
                changes.Add(new FieldChangeDetail { Field = $"处方{prescriptionIndex+1}药材", ChangeType = "新增", OldValue = "", NewValue = $"{newHerb.Name} {newHerb.Dosage}" });
            }
        }
        
        // 检查删除药材
        foreach (var oldHerb in oldHerbs ?? new List<HerbItem>()) 
        {
            if (!newHerbDict.ContainsKey(oldHerb.Name))
            {
                changes.Add(new FieldChangeDetail { Field = $"处方{prescriptionIndex+1}药材", ChangeType = "删除", OldValue = $"{oldHerb.Name} {oldHerb.Dosage}", NewValue = "" });
            }
        }
        
        // 检查剂量变更
        foreach (var newHerb in newHerbs ?? new List<HerbItem>()) 
        {
            if (oldHerbDict.ContainsKey(newHerb.Name) && oldHerbDict[newHerb.Name].Dosage != newHerb.Dosage)
            {
                changes.Add(new FieldChangeDetail { Field = $"处方{prescriptionIndex+1}药材剂量", ChangeType = "修改", OldValue = $"{newHerb.Name} {oldHerbDict[newHerb.Name].Dosage}", NewValue = $"{newHerb.Name} {newHerb.Dosage}" });
            }
        }
        
        return changes;
    }
}

    public string OldValue { get; set; } = string.Empty;
    public string NewValue { get; set; } = string.Empty;
    public string Description { get; set; } = string.Empty;
```

```csharp
public class VersionDescriptionGenerator
{
    public string GenerateAutoDescription(VersionStatus changeType, List<ChangeDetail> changes)
    {
        var summary = new StringBuilder();
        
        summary.AppendLine($"变更类型：{GetChangeTypeDescription(changeType)}");
        summary.AppendLine($"修改时间：{DateTime.Now:yyyy-MM-dd HH:mm:ss}");
        
        if (changes.Any())
        {
            summary.AppendLine("主要变更内容：");
            foreach (var change in changes.Take(3)) // 只显示前3项主要变更

                summary.AppendLine($"- {change.Field}: {change.Description}");

        }
        
        return summary.ToString();
    }
    
    private string GetChangeTypeDescription(VersionStatus type)

        return type switch
        {
            VersionStatus.Archived => "历史版本归档",
            VersionStatus.Active => "当前激活版本",
            VersionStatus.Draft => "草稿版本",
```

#### 2.2.2 版本内容范围 - 基于AI提示词标准结构
版本快照必须完整保存以下结构化数据，严格对应AI提示词输出格式：

**核心数据结构** (对应AI提示词输出JsonSchema):
- **herbal_prescriptions**: 中药处方列表
  - 包含prescription_id, name, category, description, herbs, usage, efficacy等完整字段
  - herbs数组包含name, dosage, unit, processing_method, notes等
- **herbal_warnings**: 中药安全警告列表
  - 包含warning_type, title, content, severity_level等
- **dietary_therapies**: 食疗方案列表
  - 包含dietary_therapy_id, name, category, ingredients, preparation, efficacy等
    - ingredients数组包含name, dosage, unit, processing_method, notes等
- **dietary_warnings**: 食疗安全警告列表
- **acupuncture_points**: 针灸穴位列表
  - 包含point_name, location, method, technique, efficacy等完整字段
- **moxibustion_points**: 艾灸穴位列表
  - 包含point_name, location, method, technique, efficacy等完整字段
- **cupping_points**: 拔罐部位列表
  - 包含area, method, efficacy等完整字段
- **lifestyle_advice**: 生活方式建议列表
  - 包含category, title, content, rationale等完整字段
- **dietary_advice**: 饮食建议列表
  - 包含category, title, dietary_principles等完整字段
- **follow_up_advice**: 随访建议列表
  - 包含follow_up_type, title, timing, purpose等完整字段

**版本控制元数据**:

### 2.5 版本回滚机制

#### 2.5.1 回滚流程 - 基于AI数据结构
1. **版本选择**：医生在历史版本中选择目标版本，系统展示该版本的完整AI数据结构（包含herbal_prescriptions、dietary_therapies等所有模块）
2. **影响分析**：
   - 生成回滚影响报告，按AI模块分类展示变更
   - 识别可能影响安全性的变更（如删除herbal_warnings中的禁忌药物警告）
   - 验证回滚后JSON结构是否符合AI提示词Schema
3. **回滚确认**：
   - 显示回滚前后数据结构对比（绿色新增、红色删除、黄色修改）
   - 要求输入回滚原因（审计要求）
   - 二次确认机制（防止误操作）
4. **执行回滚**：
   - 创建当前版本备份快照
   - 按AI模块顺序执行结构化回滚（先herbal_prescriptions，再acupuncture_points等）
   - 实时验证回滚进度，确保数据完整性
5. **结果验证**：
   - 验证回滚后数据符合AI提示词Schema定义
   - 验证关键安全字段（如herbal_warnings、dietary_warnings）完整性
   - 生成回滚完成报告并通知相关医生

#### 2.5.2 回滚限制与策略
- **定稿限制**：已标记为`Finalized`的版本不能回滚
- **权限限制**：只有创建人或上级医生可执行回滚
- **时间限制**：超过30天的版本需要特殊审批
- **模块化回滚**：支持按AI定义模块选择性回滚（可单独回滚中药处方、针灸穴位等）
- **数据验证**：回滚后自动验证JSON结构完整性和必填字段

```csharp
public class VersionHighRiskValidator
{
    public bool ShouldTriggerVersionVerification(TreatmentVersion targetVersion, string operationType)
    {
        switch (operationType)
        {
            case "ROLLBACK":
                return HasHighRiskContent(targetVersion.ContentSnapshot);
            case "FORCE_FINALIZE":
                return true; // 强制定稿总是需要验证
            case "DELETE":
                return targetVersion.IsFinalized && targetVersion.CreatedAt > DateTime.UtcNow.AddDays(-30);
            default:
                return false;
        }
    }
    
    private bool HasHighRiskContent(string contentSnapshot)
    {
        var content = JsonSerializer.Deserialize<TreatmentContent>(contentSnapshot);
        return content.HerbalWarnings?.Any(w => w.SeverityLevel == "严重") == true ||
               content.DietaryWarnings?.Any(w => w.SeverityLevel == "严重") == true;
    }
}
```

```csharp
[ApiController]
[Route("api/version-high-risk")]
public class VersionHighRiskVerificationController : ControllerBase
{
    [HttpPost("verify")]
    public async Task<IActionResult> VerifyVersionOperation([FromBody] VersionHighRiskVerificationRequest request)
    {
        try
        {
            // 1. 验证医生身份
            var doctor = await _verificationService.VerifyDoctorPasswordAsync(
                request.DoctorId, request.DoctorPassword);
                
            if (doctor == null)
            {
                await _auditLogService.LogFailedVersionVerificationAsync(request, "密码错误");
                return BadRequest(new { success = false, message = "医生密码验证失败" });
            }
            
            // 2. 验证操作权限
            var hasPermission = await _versionService.CheckVersionPermissionAsync(
                request.VersionId, doctor.Id, request.OperationType);
                
            if (!hasPermission)
            {
                await _auditLogService.LogFailedVersionVerificationAsync(request, "权限不足");
                return BadRequest(new { success = false, message = "操作权限不足" });
            }
            
            // 3. 创建验证记录（待审核状态）
    var verificationRecord = await _verificationService.CreateVersionVerificationRecordAsync(
        new DoctorVerificationRecord
        {
            DoctorId = doctor.Id,
            DoctorName = doctor.Name,
            TreatmentId = request.TreatmentId,
            OperationType = $"VERSION_{request.OperationType}",
            RiskLevel = DetermineRiskLevel(request.OperationType),
            RiskContext = request.RiskContext,
            ConfirmationReason = request.OperationReason,
            VerificationMethod = "ClinicReview",
            VerificationStatus = VerificationStatus.PendingReview,
            ReviewStatus = ReviewStatus.Pending, // 新增：待审核状态
            IpAddress = Request.HttpContext.Connection.RemoteIpAddress?.ToString(),
            UserAgent = Request.Headers["User-Agent"].ToString()
        });
    
    // 4. 发送实时审核通知
    await _notificationService.NotifyReviewersAsync(new ReviewNotification
    {
        TreatmentId = request.TreatmentId,
        DoctorId = doctor.Id,
        RiskLevel = DetermineRiskLevel(request.OperationType),
        VerificationRecordId = verificationRecord.Id,
        OperationType = $"版本管理-{request.OperationType}",
        ClinicalReason = request.OperationReason,
        CreatedAt = DateTime.UtcNow
    });
            
            // 4. 记录版本审计日志
            await _auditLogService.LogVersionHighRiskVerificationAsync(
                verificationRecord.Id, request, doctor);
            
            return Ok(new { 
                success = true, 
                verificationId = verificationRecord.Id,
                message = "版本操作验证成功"
            });
        }
        catch (Exception ex)
        {
            await _auditLogService.LogVersionVerificationErrorAsync(request, ex.Message);
            return StatusCode(500, new { success = false, message = "验证服务异常" });
        }
    }
}
```

```csharp
public class VersionPermissionService : IVersionPermissionService
{
    public async Task<bool> CheckVersionPermissionAsync(int versionId, int doctorId, string operationType)
    {
        var version = await _repository.GetVersionAsync(versionId);
        var doctor = await _userRepository.GetDoctorAsync(doctorId);
        
        return operationType switch

            "DELETE" => doctor.RoleLevel >= 4, // 主任医师

        };
    }
```
