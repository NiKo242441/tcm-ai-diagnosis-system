# 开发计划2：AI智能治疗方案生成（最终版 v4.0）

## 🎯 核心定位声明

**本模块为AI全自动治疗方案生成系统，仅负责展示AI生成的完整治疗方案，不涉及任何手动编辑功能。**

- ✅ **全自动**：AI根据患者完整诊疗信息智能生成治疗方案
- ✅ **零操作**：用户无需任何手动操作，系统自动完成检测、生成、保存和展示
- ✅ **专业输出**：提供符合中医临床标准的规范化医疗级治疗方案
- ✅ **安全可控**：所有内容经过AI风险识别和二次优化处理，确保100%安全可靠

---

## 1. 项目背景与目标

### 1.1 项目背景
中医数字化诊疗平台致力于通过现代信息技术提升中医诊疗的标准化和智能化水平。**治疗方案生成模块作为平台的核心AI引擎，专注于通过DifyAPI技术全自动生成完整的治疗方案。**

本模块采用**零操作设计哲学**：医生进入患者治疗方案页面时，系统自动检测治疗方案状态，未生成则立即调用AI生成并保存，已存在则直接展示，全程无需任何手动干预。所有治疗内容均由AI基于患者完整诊疗信息智能生成，确保治疗方案的专业性、个性化和安全性。

### 1.2 核心目标
- **AI治疗方案全自动生成**：基于患者完整诊疗信息（包括证候、症状、体质、病史等），通过DifyAPI智能生成涵盖中药、针灸、拔罐、艾灸、推拿、食疗六大治疗体系的完整治疗方案
- **零用户交互设计**：系统进入页面时自动检测治疗方案状态，未生成则立即调用AI生成并自动保存，已存在则直接展示，全程无需用户任何操作
- **智能风险管控**：AI自动识别并处理潜在风险内容，通过风险识别和二次优化双重过滤机制，确保展示方案100%安全可靠
- **标准化医疗输出**：提供符合中医临床标准的规范化医疗级治疗方案文档

## 2. 功能范围与优先级（AI全自动模式专用）

### 2.1 核心AI功能（零操作实现）

| 功能类别 | AI实现方式 | 用户交互 | 技术特征 |
|---------|------------|----------|----------|
| **治疗方案全自动生成** | DifyAPI智能调用 | 零操作 | 基于患者完整信息自动生成 |
| **六大治疗体系覆盖** | AI统一规划 | 零操作 | 中药+针灸+拔罐+艾灸+推拿+食疗 |
| **状态智能检测** | 自动数据库查询 | 零操作 | 进入页面即检测 |
| **风险双重过滤** | AI风险识别+二次优化 | 零操作 | 100%安全方案展示 |
| **数据自动持久化** | 后端自动保存 | 零操作 | 生成即保存，无需确认 |

### 2.2 明确排除功能（零手动干预）

| 排除功能 | 排除原因 | 负责模块 | 用户影响 |
|---------|----------|----------|----------|
| **任何手动编辑** | 违背AI全自动理念，本模块仅展示AI生成结果 | 后续编辑模块 | 页面无编辑按钮、无编辑入口 |
| **患者信息验证** | 非治疗方案职责，由患者管理模块负责数据完整性 | 患者管理模块 | 自动获取已验证的患者信息 |
| **证候确认检查** | 前置诊断流程，由证候诊断模块确保诊断准确性 | 证候诊断模块 | 自动接收已确认的证候信息 |
| **版本控制管理** | 后续流程处理，由版本管理模块自动处理 | AI版本管理模块 | 系统自动维护版本历史 |
| **手动保存确认** | 违背零操作理念，系统自动保存无需确认 | 后端自动处理 | 无保存按钮，生成即保存 |

### 2.3 AI功能边界声明

**本模块严格遵循**以下边界原则：
- ✅ **仅展示AI生成结果**：本模块仅负责展示AI生成的完整治疗方案，不提供任何内容编辑功能
- ✅ **零用户操作需求**：用户无需任何点击、输入、确认等操作，系统自动完成全流程
- ✅ **全自动流程执行**：从状态检测到方案生成、保存、展示，全流程自动化完成
- ❌ **无手动编辑入口**：页面不包含任何编辑按钮、编辑框或修改入口
- ❌ **无用户确认环节**：生成方案自动保存，无需用户点击保存或确认按钮

### 2.4 模块间接口定义（系统级标准 v4.0）

#### 2.4.1 与计划3的接口标准（强制规范）
- **状态标记**：AI生成的治疗方案必须使用`TreatmentStatus.AIGenerated`（计划1统一枚举第1项）
- **版本初始化**：必须使用`V1.0.0`作为初始版本号（计划1统一版本格式）
- **数据传递**：使用标准`TreatmentDto`对象，包含完整患者信息和AI生成内容
- **触发机制**：医生进入治疗方案页面时，系统自动检测`TreatmentStatus.AIGenerated`状态

#### 2.4.2 状态流转机制（与计划1完全统一）
```csharp
// AI生成完成后的标准状态设置（强制遵循计划1规范）
newTreatment.Status = TreatmentStatus.AIGenerated;  // 计划1统一枚举值1
newTreatment.IsAiOriginated = true;                 // 强制标准字段
newTreatment.Version = "V1.0.0";                  // 计划1统一版本格式
```


#### 2.4.4 异常处理标准
- **AI生成失败**：保持数据库状态不变，不创建治疗方案记录
- **状态回滚**：如生成过程中断，确保不产生AIGenerated状态的脏数据
- **日志记录**：所有AI调用必须记录到计划6的审计日志中
- **用户提示**：提供"AI生成失败，请联系技术支持"的标准化错误信息

#### 2.4.5 与后续模块的边界声明（系统级标准 v4.0）
- **计划3职责**：仅接收AIGenerated(1)状态的治疗方案进行编辑，版本为V1.0.0
- **计划4职责**：仅处理Editing(2)状态的治疗方案进行禁忌检查
- **计划5职责**：仅处理Versioning(5)状态的治疗方案进行版本管理，版本号遵循V{主}.{次}.{修订}格式
- **计划6职责**：记录所有AI生成操作的完整审计轨迹，包含AI模型版本信息
- **计划7职责**：仅处理Finalized(6)状态的治疗方案进行打印导出
- **数据完整性**：生成的治疗方案必须包含六大治疗体系的完整数据结构（与计划1实体类匹配）

## 3. 技术实现方案

### 3.1 AI全自动治疗方案展示系统设计

#### 3.1.1 零操作业务流程

**AI全自动业务流程**（用户视角）：
```
医生进入治疗方案页面 → 系统自动展示完整AI治疗方案 → 医生查看完成
```

**系统内部执行流程**：
```
页面加载 → 自动检测治疗方案状态 → 
[未生成]调用DifyAPI生成方案 → 自动保存到数据库 → 展示新生成的方案 →
[已生成]直接从数据库获取 → 展示已有方案 → 医生查看完成
```

**AI模式状态定义**：
- **AI展示模式**：仅展示AI生成的完整治疗方案，无任何编辑功能（默认模式）
- **AI打印模式**：专业医疗文书格式输出，包含AI生成标识和防伪信息
- **AI安全模式**：经过AI风险识别和二次优化双重过滤的100%安全治疗方案展示

```javascript
// 检查治疗方案状态
async function checkTreatmentStatus(visitId) {
    try {
        const response = await fetch(`/Doctor/Treatments/CheckStatus?visitId=${visitId}`);
        const data = await response.json();
        
        if (!data.hasTreatment) {
            // 自动触发AI生成
            await generateAITreatment(visitId);
        } else {
            // 已有治疗方案，直接展示
            console.log('治疗方案已存在，直接展示');
        }
    } catch (error) {
        console.error('检查治疗方案状态时出错:', error);
    }
}

// AI治疗方案生成（自动触发，无需用户确认）
async function generateAITreatment(visitId) {
    try {
        showLoading();
        
        const response = await fetch(`/Doctor/Treatments/GenerateAI?visitId=${visitId}`, {
            method: 'POST',
            headers: {
                'Content-Type': 'application/json',
                'RequestVerificationToken': document.querySelector('input[name="__RequestVerificationToken"]')?.value
            }
        });
        
        const data = await response.json();
        if (data.success) {
            console.log('AI治疗方案已自动生成');
            location.reload(); // 刷新页面显示新生成的方案
        } else {
            console.error('AI生成失败:', data.message);
            showErrorMessage('AI生成治疗方案失败，请稍后重试');
        }
    } catch (error) {
        console.error('AI生成过程中发生错误:', error);
        showErrorMessage('生成过程中发生错误，请稍后重试');
    }
}

    const alertDiv = document.createElement('div');

        <button type="button" class="btn-close" data-bs-dismiss="alert"></button>

}
```

### 3.3 DifyAPI集成方案（统一标准调用）

#### 3.3.1 工作流架构设计

**核心工作流**：
```
患者就诊信息 → 证候识别 → 治疗方案生成 → 禁忌症检查 → 风险评估 → 个性化调整 → 最终方案
```

**工作流组件**：
- **证候分析模块**：基于症状、舌象、脉象进行证候识别
- **方案生成模块**：根据证候类型生成对应治疗方案
- **禁忌检查模块**：检查药物配伍禁忌、特殊人群禁忌
- **风险评估模块**：评估治疗方案的潜在风险
- **个性化调整模块**：根据患者个体情况进行方案微调

#### 3.3.2 Dify工作流集成说明

**工作流调用规范**

本模块直接使用Dify内置的治疗方案生成工作流，无需配置工作流参数和提示词模板。调用规范遵循 `docs/DifyAPI.md` 中的标准API接口规范。

**输入输出规范**

输入输出规范参考 `docs\AI提示词\治疗方案\输入输出.md` 文件中的定义，包括：
- 诊断信息（证候、症状、舌象、脉象等）
- 患者基本信息（年龄、性别、体质等）
- 治疗偏好设置

所有参数传递和响应处理均通过标准Dify API接口完成，无需额外的配置工作。

```csharp
public interface IDifyTreatmentService
{
    Task<DifyTreatmentResponse> GenerateTreatmentAsync(TreatmentGenerationRequest request);
    Task<DifyTreatmentResponse> RegenerateTreatmentAsync(int treatmentId, string modifications);
    Task<bool> ValidateTreatmentAsync(TreatmentValidationRequest request);
}

public class DifyTreatmentService : IDifyTreatmentService
{
    private readonly HttpClient _httpClient;
    private readonly ILogger<DifyTreatmentService> _logger;
    private readonly IConfiguration _configuration;
    
    public async Task<DifyTreatmentResponse> GenerateTreatmentAsync(TreatmentGenerationRequest request)
    {
        try
        {
            var apiUrl = _configuration["Dify:ApiUrl"];
            var apiKey = _configuration["Dify:ApiKey"];
            var workflowId = _configuration["Dify:TreatmentWorkflowId"];
            
            var payload = new
            {
                inputs = new
                {
                    patient_info = BuildPatientInfo(request.PatientId),
                    confirmed_syndrome = request.ConfirmedSyndrome,
                    medical_history = BuildMedicalHistory(request.PatientId)
                },
                user = request.CreatedByUserId.ToString(),
                conversation_id = request.ConversationId
            };
            
            var json = JsonSerializer.Serialize(payload);
            var content = new StringContent(json, Encoding.UTF8, "application/json");
            
            _httpClient.DefaultRequestHeaders.Authorization = 
                new AuthenticationHeaderValue("Bearer", apiKey);
            
            var response = await _httpClient.PostAsync($"{apiUrl}/workflows/{workflowId}/run", content);
            response.EnsureSuccessStatusCode();
            
            var responseContent = await response.Content.ReadAsStringAsync();
            return JsonSerializer.Deserialize<DifyTreatmentResponse>(responseContent);
        }
        catch (Exception ex)
        {
            _logger.LogError(ex, "Failed to generate treatment plan via Dify API");
            throw new TreatmentGenerationException("AI治疗方案生成失败，请稍后重试", ex);
        }
    }
    
    private object BuildPatientInfo(int patientId)
    {
        // 从数据库获取患者信息并构建请求对象
        var patient = _patientService.GetPatientById(patientId);
        return new
        {
            basic = new
            {
                name = patient.Name,
                age = patient.Age,
                gender = patient.Gender,
                constitution = patient.ConstitutionType
            },
            symptoms = GetLatestSymptoms(patientId),
            examination = GetLatestExamination(patientId)
        };
    }
}
```

```csharp
    private readonly IDifyApiService _difyService;
    private readonly IRiskCheckService _riskCheckService;
    private readonly INamingValidator _namingValidator;

    public TreatmentGenerationService(
        IDifyApiService difyService,
        IRiskCheckService riskCheckService,
        INamingValidator namingValidator,

        _difyService = difyService;
        _riskCheckService = riskCheckService;
        _namingValidator = namingValidator;
        _logger = logger;

    public async Task<ApiResponse<TreatmentEditDto>> GenerateTreatmentAsync(TreatmentGenerationRequestDto request)
    {
        try
        {
            // 1. 调用Dify API生成治疗方案
            var aiResponse = await _difyService.GenerateTreatmentAsync(request);
            if (!aiResponse.Success)
            {
                return new ApiResponse<TreatmentEditDto>
                {
                    Success = false,
                    Message = "AI生成失败",
                    Errors = new List<string> { aiResponse.ErrorMessage }
                };
            }

            // 2. 解析AI响应
            var treatment = ParseAiResponse(aiResponse.Data);
            
            // 3. 标准化命名
            await StandardizeNamesAsync(treatment);
            
            return new ApiResponse<TreatmentEditDto>
            {
                Success = true,
                Message = "治疗方案生成成功",
                Data = treatment
            };
        }
        catch (Exception ex)
        {
            _logger.LogError(ex, "治疗方案生成失败");
            return new ApiResponse<TreatmentEditDto>
            {
                Success = false,
                Message = "系统内部错误",
                Errors = new List<string> { ex.Message }
            };
        }
```

```csharp
public class HighRiskVerificationService : IHighRiskVerificationService

    {
        // 1. 验证医生身份
        var isValidDoctor = await ValidateDoctorPasswordAsync(request.DoctorId, request.Password);
        if (!isValidDoctor)
            return VerificationResult.Failed("医生密码验证失败");

        var verificationRecord = new DoctorVerificationRecord
        {
            DoctorId = request.DoctorId,
            TreatmentId = request.TreatmentId,
            RiskLevel = request.RiskLevel,

            VerificationStatus = VerificationStatus.Approved,
            VerificationReason = request.Reason,

            CreatedAt = DateTime.UtcNow,
            IpAddress = request.IpAddress,
            UserAgent = request.UserAgent
        };

        await _treatmentRepository.SaveVerificationRecordAsync(verificationRecord);

        // 4. 记录审计日志
        await _auditLogService.LogHighRiskVerificationAsync(verificationRecord);

        return VerificationResult.Success(verificationRecord.Id);
    }
```

   ```csharp
   private async Task<AiTreatmentResponse> HandleRiskAssessmentAsync(AiTreatmentResponse response, TreatmentGenerationRequestDto originalRequest)
   {
       if (response.RiskWarnings?.Any() != true)
           return response;
       
       // 如果存在风险，将风险信息重新发送给Dify API
       var retryRequest = new TreatmentGenerationRequestDto
       {
           VisitId = originalRequest.VisitId,
           SyndromeId = originalRequest.SyndromeId,
           PatientInfo = originalRequest.PatientInfo,
           SyndromeAnalysis = originalRequest.SyndromeAnalysis,
           Symptoms = originalRequest.Symptoms,
           TreatmentPreference = originalRequest.TreatmentPreference,
           // 添加风险信息作为上下文
           Context = string.Join(";", response.RiskWarnings.Select(w => w.Message))
       };
       
       return await CallAiGenerationAsync(retryRequest);
   }
   ```

```typescript
        try {
            // 1. 检查是否已存在治疗方案
            const existingTreatment = await api.get(`/api/treatments/visit/${visitId}`);
```
