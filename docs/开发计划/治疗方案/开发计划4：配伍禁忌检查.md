# 开发计划4：配伍禁忌检查（中医专用版）

## 1. 功能定位与业务边界

配伍禁忌检查是治疗方案模块中的**强制性安全检查环节**，位于"治疗方案内容管理与编辑"阶段之后，"治疗方案版本管理"阶段之前。该功能确保在医生提交治疗方案定稿前，系统能够自动识别并提示所有可能的中医药安全风险。

**前置条件**：
- 医生已完成对AI生成治疗方案的编辑和调整
- 治疗方案处于草稿状态（IsArchived = false）
- 患者基本信息完整（年龄、性别、妊娠状态等中医辨证要素）

**后置处理**：
- 检查结果直接影响治疗方案的定稿流程
- 所有风险确认操作需记录审计日志
- 风险确认信息成为病历永久组成部分

### 1.1 功能边界声明（系统级标准 v4.0）

**本模块严格遵循**以下边界原则：
- ✅ **仅处理Editing(2)状态的治疗方案**：仅接收计划3输出的Editing状态治疗方案
- ✅ **提供实时检查反馈**：即时显示检查结果和建议
- ✅ **支持多维度检查**：涵盖十八反、十九畏、药性冲突、特殊人群禁忌、六大治疗体系完整性验证
- ✅ **状态流转管理**：检查完成后自动更新治疗方案状态为Versioning(5)或CheckFailed(4)
- ✅ **版本规范**：检查通过后自动触发计划5的版本管理流程
- ❌ **不修改治疗方案内容**：仅提供检查结果和建议，不主动修改方案
- ❌ **不处理版本号生成**：版本号由计划5统一处理（从V1.0.0开始递增）
- ❌ **不处理打印导出**：打印功能由计划7统一处理

### 1.2 状态流转机制（系统级标准 v4.0）

#### 1.2.1 检查触发条件（强制规范）
- **状态限制**：仅处理`TreatmentStatus.Editing`状态的治疗方案
- **前置条件**：必须完成计划3的医生编辑阶段
- **数据完整性**：患者基本信息必须完整（年龄、性别、妊娠状态等）
- **版本要求**：必须基于最新版本（IsLatest = true）进行检查

#### 1.2.2 标准状态流转机制（与计划1完全统一）
```csharp
// 检查完成后的标准状态更新（强制遵循计划1规范）
if (checkResult.IsPassed)
{
    treatment.Status = TreatmentStatus.Versioning;  // 计划1统一枚举值5
    treatment.LastCheckedAt = DateTime.UtcNow;
    treatment.CheckedByUserId = currentUserId;
    // 自动触发计划5版本管理模块，版本号由计划5统一处理
}
else
{
    treatment.Status = TreatmentStatus.CheckFailed;  // 计划1统一枚举值4
    treatment.CheckFailureReason = checkResult.FailureReason;
    // 返回计划3重新编辑，版本号保持不变
}
```

#### 1.2.3 状态流转详细规范
| 当前状态 | 触发条件 | 检查结果 | 下一状态 | 负责模块 | 备注 |
|----------|----------|----------|----------|----------|------|
| Editing | 医生完成编辑 | 检查通过 | Versioning | 计划5 | 自动生成新版本 |
| Editing | 医生完成编辑 | 检查失败 | CheckFailed | 计划3 | 返回重新编辑 |
| CheckFailed | 医生修改完成 | 重新检查 | Checking | 计划4 | 再次进行禁忌检查 |

#### 1.2.4 与计划3/5/6的接口标准
- **接收数据**：仅接收计划3输出的Editing状态治疗方案
- **输出数据**：仅向计划5输出Versioning或CheckFailed状态
- **审计记录**：所有检查结果必须记录到计划6的审计日志中
- **版本管理**：不处理版本号，由计划5统一处理版本管理

#### 1.2.5 检查完成后的标准处理
- **检查通过**：自动触发计划5的版本管理流程
- **检查失败**：提供详细失败原因，返回计划3重新编辑
- **审计记录**：记录检查时间、检查人员、检查结果等完整信息
- **数据锁定**：检查期间锁定治疗方案，防止并发修改

## 2. 核心业务规则

### 2.1 检查维度与优先级

| 检查类型 | 风险等级 | 处理方式 | 业务规则 |
|----------|----------|----------|----------|
| **绝对禁忌** | 高风险 | 禁止定稿 | 必须修改后才能提交 |
| **相对禁忌** | 中风险 | 二次确认 | 需医生勾选确认并记录原因 |
| **注意事项** | 低风险 | 提示提醒 | 显示建议但不阻止提交 |

### 2.2 传统配伍禁忌检查

**十八反检查规则**：
- **甘草反**：甘草与甘遂、大戟、芫花、海藻
- **乌头反**：乌头与贝母、瓜蒌、半夏、白蔹、白及
- **藜芦反**：藜芦与人参、沙参、丹参、玄参、细辛、芍药

**十九畏检查规则**：
- 硫黄畏朴硝，水银畏砒霜，狼毒畏密陀僧
- 巴豆畏牵牛，丁香畏郁金，牙硝畏三棱
- 川乌、草乌畏犀角，人参畏五灵脂
- 官桂畏赤石脂

**经方配伍禁忌**：
- 麻黄与桂枝同用的禁忌情况
- 石膏与附子的配伍限制
- 大黄与附子的同用条件

**业务处理**：
- 发现反畏组合时标记为"绝对禁忌"
- 提供替代药材建议（如以太子参替代人参配藜芦）
- 记录检查依据和经典文献出处（《本草纲目》、《神农本草经》等）

### 2.3 中药药性冲突检查

**四气五味冲突**：
- 寒凉药材与温热药材的过度配伍
- 五味偏胜的组合（如过酸、过苦）
- 升降浮沉的相反作用

**归经冲突**：
- 同一归经中作用相反的药材
- 归经过于分散导致疗效减弱
- 归经与病位不符的情况

**毒性分级管理**：
- **剧毒**：砒霜、水银、马钱子等（绝对禁用）
- **大毒**：附子、乌头、巴豆等（严格限量）
- **小毒**：半夏、南星、全蝎等（注意监控）

**检查要点**：
- 毒性药物剂量是否超过《药典》规定
- 炮制方法是否适当（如附子是否先煎）
- 是否有必要的解毒配伍（如甘草解附子毒）

### 2.4 特殊人群禁忌检查

| 人群类型 | 中医辨证重点 | 禁忌内容 | 禁忌级别 |
|----------|--------------|----------|----------|
| **孕妇** | 胎元稳固、气血调和 | 活血化瘀药、泻下药、滑利药 | 绝对禁忌 |
| **哺乳期** | 乳汁生化、脾胃运化 | 回乳药、燥烈药 | 相对禁忌 |
| **儿童** | 稚阴稚阳、脾常不足 | 峻补药、攻伐药 | 相对禁忌 |
| **老年人** | 肾精亏虚、脾胃虚弱 | 滋腻药、峻猛药 | 注意事项 |
| **体虚者** | 气血阴阳偏衰 | 发散药、攻邪药 | 相对禁忌 |

**妊娠禁忌药物**：
- **禁用**：麝香、三棱、莪术、水蛭、虻虫
- **慎用**：桃仁、红花、大黄、枳实、附子
- **特殊处理**：安胎药与治病药的平衡

### 2.5 证候用药禁忌检查

**辨证禁忌**：
- 虚证误用泻法
- 实证误用补法
- 寒证误用热性药
- 热证误用热性药

**脏腑辨证**：
- 脾虚忌用滋腻药
- 肾虚忌用克伐药
- 肝旺忌用升发药
- 肺燥忌用辛燥药

### 2.6 儿童用药禁忌规则库

**年龄分级禁忌**：
- **新生儿期**（0-28天）：禁用所有泻下药、活血化瘀药
- **婴儿期**（28天-1岁）：慎用苦寒药、慎用有毒药材
- **幼儿期**（1-3岁）：减量使用滋补药、禁用峻下逐水药
- **学龄前期**（3-6岁）：慎用安神药、禁用毒性药材
- **学龄期**（6-12岁）：按体重调整剂量、慎用激素类药材

**儿童专用剂量算法**：
```csharp
public class PediatricDosageCalculator
{
    /// <summary>
    /// 计算儿童安全剂量
    /// </summary>
    /// <param name="herbName">药材名称</param>
    /// <param name="patientInfo">患者信息</param>
    /// <param name="adultDosage">成人用量</param>
    /// <returns>儿童安全用量</returns>
    public decimal CalculateSafeDosage(string herbName, PatientInfo patientInfo, decimal adultDosage)
    {
        int age = patientInfo.Age;
        
        // 基于年龄的儿童剂量计算（遵循传统中医儿科用药原则）
        if (age < 1) return adultDosage * 0.1m;      // 新生儿期
        if (age < 3) return adultDosage * 0.2m;      // 婴儿期
        if (age < 6) return adultDosage * 0.3m;      // 幼儿期
        if (age < 12) return adultDosage * 0.5m;     // 学龄期
        return adultDosage * 0.8m;                   // 青少年期
    }

    /// <summary>
    /// 检查儿童用药禁忌
    /// </summary>
    /// <param name="herbName">药材名称</param>
    /// <param name="patientInfo">患者信息</param>
    /// <returns>禁忌信息，如无禁忌返回null</returns>
    public HerbalWarning CheckPediatricContraindication(string herbName, PatientInfo patientInfo)
    {
        int age = patientInfo.Age;
        string gender = patientInfo.Gender;
        
        // 新生儿期禁忌
        if (age < 28)
        {
            if (IsPurgativeHerb(herbName) || IsBloodActivatingHerb(herbName))
            {
                return new HerbalWarning
                {
                    WarningType = "儿童用药禁忌",
                    Title = "新生儿禁用药物",
                    Content = $"{herbName}为泻下药或活血化瘀药，新生儿禁用",
                    SeverityLevel = "严重",
                    AffectedMedications = new List<string> { herbName },
                    SpecialPopulations = "新生儿期（0-28天）",
                    ClassicalSource = "《小儿药证直诀》",
                    TraditionalRationale = "新生儿脏腑娇嫩，禁用攻伐之品"
                };
            }
        }

        // 婴儿期禁忌
        if (age < 1)
        {
            if (IsBitterColdHerb(herbName) || IsToxicHerb(herbName))
            {
                return new HerbalWarning
                {
                    WarningType = "儿童用药禁忌",
                    Title = "婴儿期慎用药物",
                    Content = $"{herbName}性苦寒或有毒，婴儿期慎用",
                    SeverityLevel = "中等",
                    AffectedMedications = new List<string> { herbName },
                    SpecialPopulations = "婴儿期（28天-1岁）",
                    ClassicalSource = "《幼幼集成》",
                    TraditionalRationale = "婴儿脾胃薄弱，慎用苦寒有毒之品"
                };
            }
        }

        return null;
    }

    // 辅助方法：判断是否为泻下药
    private bool IsPurgativeHerb(string herbName)
    {
        string[] purgativeHerbs = { "大黄", "芒硝", "番泻叶", "芦荟", "巴豆" };
        return purgativeHerbs.Contains(herbName);
    }

    // 辅助方法：判断是否为活血化瘀药
    private bool IsBloodActivatingHerb(string herbName)
    {
        string[] bloodActivatingHerbs = { "桃仁", "红花", "三棱", "莪术", "水蛭", "虻虫" };
        return bloodActivatingHerbs.Contains(herbName);
    }

    // 辅助方法：判断是否为苦寒药
    private bool IsBitterColdHerb(string herbName)
    {
        string[] bitterColdHerbs = { "黄连", "黄芩", "黄柏", "栀子", "龙胆草" };
        return bitterColdHerbs.Contains(herbName);
    }

    // 辅助方法：判断是否为有毒药材
    private bool IsToxicHerb(string herbName)
    {
        string[] toxicHerbs = { "附子", "乌头", "马钱子", "半夏", "南星" };
        return toxicHerbs.Contains(herbName);
    }
}
```

### 2.7 动态规则更新机制

**规则版本管理**：
- **规则库版本控制**：每次更新都生成新版本，支持回滚
- **生效时间控制**：可设置规则的生效时间和适用范围
- **灰度发布**：新规则可先在小范围试用，稳定后全面推广
- **影响评估**：更新前评估对现有治疗方案的影响

**剂量阈值动态调整**：
- **基于临床数据**：根据实际用药效果和不良反应数据调整阈值
- **季节性调整**：考虑不同季节的用药特点调整禁忌规则
- **地域性差异**：支持不同地区的用药习惯差异
- **实时更新API**：提供规则实时更新接口，无需重启系统

3. 提供符合中医理论的修改建议

1. 展示当前证候分析

## 6. 中医审计与合规要求

```csharp
public class ContraindicationRiskAssessmentService : IContraindicationRiskService
{
    public async Task<ContraindicationRiskResult> AssessContraindicationRiskAsync(
        TreatmentPlan plan, List<HerbalWarning> warnings)
    {
        var riskScore = 0;
        var criticalWarnings = warnings.Where(w => w.RiskLevel == RiskLevel.Critical);
        
        // 1. 经典禁忌严重程度评估
        foreach (var warning in criticalWarnings)
        {
            riskScore += warning.WarningType switch
            {
                "十八反" => 8,      // 经典绝对禁忌
                "十九畏" => 7,      // 经典相对禁忌
                "妊娠禁忌" => 10,   // 特殊人群绝对禁忌
                "毒性超限" => 9,    // 安全剂量超限
                "儿童禁忌" => 8,    // 年龄禁忌
                _ => 5
            };
        }
        
        return new ContraindicationRiskResult
        {
            RiskLevel = DetermineRiskLevel(riskScore),
            CriticalWarnings = criticalWarnings.ToList(),
            RequiresVerification = riskScore >= 7,
            VerificationMethod = DetermineVerificationMethod(riskScore),
            ClassicalReferences = ExtractClassicalReferences(warnings)
        };
    }
    
    private RiskLevel DetermineRiskLevel(int score)
    {
        return score switch
        {
            >= 15 => RiskLevel.Critical,   // 极高风险，需多级验证
            >= 10 => RiskLevel.High,      // 高风险，需密码验证
            >= 7 => RiskLevel.Medium,     // 中风险，需确认对话框
            _ => RiskLevel.Low            // 低风险，直接提示
        };
    }
}
```

```csharp
public class TcmContraindicationVerificationService : ITcmContraindicationService
{
    public async Task<ContraindicationVerificationResult> ProcessContraindicationVerificationAsync(
        ContraindicationVerificationRequest request)
    {
        var assessment = await _riskService.AssessContraindicationRiskAsync(
            request.TreatmentPlan, request.Warnings);
        
        switch (assessment.RiskLevel)
        {
            case RiskLevel.Low:
                return new ContraindicationVerificationResult { Approved = true };
                
            case RiskLevel.Medium:
                return await ProcessMediumRiskContraindication(request, assessment);
                
            case RiskLevel.High:
                return await ProcessHighRiskContraindication(request, assessment);
                
            case RiskLevel.Critical:
                return await ProcessCriticalRiskContraindication(request, assessment);
                
            default:
                throw new ArgumentOutOfRangeException();
        }
    }

    private async Task<ContraindicationVerificationResult> ProcessHighRiskContraindication(
        ContraindicationVerificationRequest request, 
        ContraindicationRiskResult assessment)
    {
        // 1. 验证医生密码
        var passwordValid = await _userService.ValidateDoctorPasswordAsync(
            request.DoctorId, request.DoctorPassword);
        
        if (!passwordValid)
            return ContraindicationVerificationResult.Failed("医生密码验证失败");

        // 2. 创建中医禁忌验证记录
        var record = new DoctorVerificationRecord
        {
            DoctorId = request.DoctorId,
            TreatmentId = request.TreatmentId,
            RiskLevel = RiskLevel.High,
            VerificationMethod = VerificationMethod.Password,
            VerificationStatus = VerificationStatus.Approved,
            OperationType = "CONTRAINDICATION_OVERRIDE",
            VerificationReason = $@"
                临床理由：{request.ClinicalReason}
                经典依据：{request.ClassicalBasis}
                涉及禁忌：{string.Join(", ", assessment.CriticalWarnings.Select(w => w.Title))}
            ",
            ClassicalReferences = JsonSerializer.Serialize(assessment.ClassicalReferences),
            CreatedAt = DateTime.UtcNow,
            IpAddress = request.IpAddress,
            UserAgent = request.UserAgent
        };

        await _verificationRepository.SaveAsync(record);
        await _auditLogService.LogContraindicationVerificationAsync(record);

        return ContraindicationVerificationResult.Success(record.Id);
    }

    private async Task<ContraindicationVerificationResult> ProcessCriticalRiskContraindication(
        ContraindicationVerificationRequest request, 
        ContraindicationRiskResult assessment)
    {
        // 1. 验证医生密码
        var passwordValid = await _userService.ValidateDoctorPasswordAsync(
            request.DoctorId, request.DoctorPassword);
        
        if (!passwordValid)
            return ContraindicationVerificationResult.Failed("医生密码验证失败");

        // 2. 创建极高风险验证记录（待审核状态）
        var record = new DoctorVerificationRecord
        {
            DoctorId = request.DoctorId,
            TreatmentId = request.TreatmentId,
            RiskLevel = RiskLevel.Critical,
            VerificationMethod = VerificationMethod.ClinicReview,
            VerificationStatus = VerificationStatus.PendingReview,
            OperationType = "CRITICAL_CONTRAINDICATION_OVERRIDE",
            VerificationReason = $@"
                中医辨证：{request.TcmAnalysis}
                经典文献：{request.LiteratureCitation}
                禁忌详情：{string.Join(", ", assessment.CriticalWarnings.Select(w => w.Title))}
            ",
            ReviewStatus = ReviewStatus.Pending, // 新增：待审核状态
            ClassicalReferences = JsonSerializer.Serialize(assessment.ClassicalReferences),
            CreatedAt = DateTime.UtcNow,
            IpAddress = request.IpAddress,
            UserAgent = request.UserAgent
        };

        await _verificationRepository.SaveAsync(record);
        
        // 3. 发送实时审核通知给诊所药剂师/管理员
        await _notificationService.NotifyReviewersAsync(new ReviewNotification
        {
            TreatmentId = request.TreatmentId,
            DoctorId = request.DoctorId,
            RiskLevel = RiskLevel.Critical,
            VerificationRecordId = record.Id,
            ClinicalReason = request.TcmAnalysis,
            OperationType = "配伍禁忌检查-极高风险",
            CreatedAt = DateTime.UtcNow
        });

        await _auditLogService.LogCriticalContraindicationVerificationAsync(record);

        return ContraindicationVerificationResult.PendingReview(record.Id, "已提交诊所药剂师/管理员审核");
    }
}
```
