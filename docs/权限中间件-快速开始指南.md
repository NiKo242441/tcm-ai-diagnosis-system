# 权限验证中间件 - 快速开始指南

## 🚀 5分钟快速上手

### 1. 确认系统已启动

权限验证中间件已经集成到系统中，启动应用程序即可使用：

```bash
cd TcmAiDiagnosis.Web
dotnet run
```

应用程序将运行在：`http://localhost:5226`

### 2. 验证权限系统是否工作

#### 方法1：访问受保护的API

访问已添加权限标注的API端点：

```bash
# 未登录访问（应返回401）
curl http://localhost:5226/api/Herb/paged

# 登录后访问（根据权限返回200或403）
curl -H "Authorization: Bearer YOUR_TOKEN" http://localhost:5226/api/Herb/paged
```

#### 方法2：查看日志

启动应用程序后，查看控制台日志，应该看到：

```
info: TcmAiDiagnosis.Web.Extensions.PermissionServiceExtensions[0]
      权限系统服务已注册
```

### 3. 为API添加权限保护

#### 步骤1：添加using语句

```csharp
using TcmAiDiagnosis.Web.Attributes;
using TcmAiDiagnosis.Domain.Constants;
```

#### 步骤2：添加权限特性

```csharp
[HttpGet]
[RequirePermission(PermissionConstants.PatientManagement.View)]
public async Task<IActionResult> GetPatients()
{
    // 方法实现
}
```

#### 步骤3：测试权限验证

重启应用程序，访问该API：
- 未登录 → 返回401
- 已登录但无权限 → 返回403
- 已登录且有权限 → 正常返回数据

## 📖 常用场景

### 场景1：单个权限

```csharp
[HttpGet]
[RequirePermission(PermissionConstants.PatientManagement.View)]
public async Task<IActionResult> GetPatient(int id)
{
    // 只有具有 patient.view 权限的用户才能访问
}
```

### 场景2：多个权限（需要全部）

```csharp
[HttpPut("{id}")]
[RequirePermission(
    PermissionConstants.PatientManagement.Edit,
    PermissionConstants.PatientManagement.View
)]
public async Task<IActionResult> UpdatePatient(int id, PatientDto dto)
{
    // 用户必须同时具有 patient.edit 和 patient.view 权限
}
```

### 场景3：多个权限（需要任一）

```csharp
[HttpGet]
[RequirePermission(
    PermissionConstants.PatientManagement.View,
    PermissionConstants.PatientManagement.Search,
    Mode = PermissionCheckMode.RequireAny
)]
public async Task<IActionResult> SearchPatients(string keyword)
{
    // 用户只需具有 patient.view 或 patient.search 任一权限即可
}
```

### 场景4：控制器级别权限

```csharp
[ApiController]
[Route("api/[controller]")]
[RequirePermission(PermissionConstants.PatientManagement.View)]
public class PatientController : ControllerBase
{
    // 控制器中的所有方法都需要 patient.view 权限
    
    [HttpGet]
    public async Task<IActionResult> GetAll() { }
    
    [HttpGet("{id}")]
    public async Task<IActionResult> GetById(int id) { }
}
```

### 场景5：方法覆盖控制器权限

```csharp
[ApiController]
[Route("api/[controller]")]
[RequirePermission(PermissionConstants.PatientManagement.View)]
public class PatientController : ControllerBase
{
    [HttpGet]
    public async Task<IActionResult> GetAll() 
    {
        // 继承控制器权限：需要 patient.view
    }
    
    [HttpPost]
    [RequirePermission(PermissionConstants.PatientManagement.Create)]
    public async Task<IActionResult> Create(PatientDto dto) 
    {
        // 方法权限覆盖控制器权限：需要 patient.create
    }
}
```

## ⚙️ 配置选项

### 启用/禁用权限检查

在 `appsettings.json` 中：

```json
{
  "PermissionSettings": {
    "EnablePermissionCheck": true  // 设置为false可禁用权限检查
  }
}
```

### 添加白名单路径

```json
{
  "PermissionSettings": {
    "WhitelistPaths": [
      "/api/auth/login",
      "/api/health",
      "/swagger",
      "/your-custom-path"  // 添加自定义白名单路径
    ]
  }
}
```

### 调整缓存过期时间

```json
{
  "PermissionSettings": {
    "CacheExpirationMinutes": 30  // 默认30分钟，可根据需要调整
  }
}
```

## 🔍 调试技巧

### 1. 查看用户权限

在业务逻辑中注入 `IPermissionService`：

```csharp
public class MyService
{
    private readonly IPermissionService _permissionService;
    
    public MyService(IPermissionService permissionService)
    {
        _permissionService = permissionService;
    }
    
    public async Task DebugUserPermissions(int userId, int? tenantId)
    {
        var permissions = await _permissionService.GetUserPermissionsAsync(
            userId, 
            tenantId, 
            includeTemporary: true
        );
        
        Console.WriteLine($"用户 {userId} 的权限：");
        foreach (var permission in permissions)
        {
            Console.WriteLine($"  - {permission}");
        }
    }
}
```

### 2. 查看权限验证日志

在 `appsettings.json` 中启用详细日志：

```json
{
  "Logging": {
    "LogLevel": {
      "Default": "Information",
      "TcmAiDiagnosis.Domain.Services": "Debug",  // 启用权限服务调试日志
      "TcmAiDiagnosis.Web.Middleware": "Debug"    // 启用中间件调试日志
    }
  }
}
```

### 3. 测试权限检查

创建测试端点：

```csharp
[ApiController]
[Route("api/[controller]")]
public class PermissionTestController : ControllerBase
{
    private readonly IPermissionService _permissionService;
    
    public PermissionTestController(IPermissionService permissionService)
    {
        _permissionService = permissionService;
    }
    
    [HttpGet("my-permissions")]
    public async Task<IActionResult> GetMyPermissions()
    {
        var userId = int.Parse(User.FindFirst(ClaimTypes.NameIdentifier)?.Value ?? "0");
        var tenantId = int.Parse(User.FindFirst("TenantId")?.Value ?? "0");
        
        var permissions = await _permissionService.GetUserPermissionsAsync(
            userId, 
            tenantId
        );
        
        return Ok(new { userId, tenantId, permissions });
    }
    
    [HttpGet("check/{permissionCode}")]
    public async Task<IActionResult> CheckPermission(string permissionCode)
    {
        var userId = int.Parse(User.FindFirst(ClaimTypes.NameIdentifier)?.Value ?? "0");
        var tenantId = int.Parse(User.FindFirst("TenantId")?.Value ?? "0");
        
        var hasPermission = await _permissionService.CheckPermissionAsync(
            userId, 
            tenantId, 
            permissionCode
        );
        
        return Ok(new { permissionCode, hasPermission });
    }
}
```

## ❓ 常见问题

### Q1: 为什么我的API总是返回401？

**A**: 检查以下几点：
1. 用户是否已登录？
2. 请求是否包含有效的认证令牌？
3. 用户声明中是否包含 `ClaimTypes.NameIdentifier`？

### Q2: 为什么我的API总是返回403？

**A**: 检查以下几点：
1. 用户是否具有所需权限？使用调试端点查看用户权限列表
2. 权限代码是否正确？检查 `PermissionConstants` 中的定义
3. 数据库中是否已正确初始化权限数据？

### Q3: 如何临时禁用权限检查？

**A**: 在 `appsettings.json` 中设置：
```json
{
  "PermissionSettings": {
    "EnablePermissionCheck": false
  }
}
```

### Q4: 如何刷新用户权限缓存？

**A**: 当用户角色或权限变更后，调用：
```csharp
await _permissionService.RefreshUserPermissionCacheAsync(userId, tenantId);
```

### Q5: 白名单路径不生效？

**A**: 检查路径格式：
- 使用 `/api/auth/login` 而不是 `api/auth/login`
- 路径匹配使用 `StartsWithSegments`，所以 `/api/auth` 会匹配 `/api/auth/login`
