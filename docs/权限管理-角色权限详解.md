# 权限管理系统 - 角色权限详解

## 概述

本文档详细说明TCM AI诊断系统中各个角色的权限及其具体作用，帮助理解权限系统的设计意图和使用场景。

## 角色体系

系统定义了4个核心角色：

| 角色 | 英文名 | 权限数量 | 主要职责 |
|------|--------|----------|----------|
| 租户管理员 | Manager | 24个 | 管理租户内的用户、配置和监控 |
| 中医师 | Doctor | 51个 | 诊疗、开方、病历管理 |
| 患者 | Patient | 11个 | 查看个人信息和诊疗记录 |
| 药剂师 | Pharmacist | 6个 | 查看处方和中药信息 |

---

## 一、租户管理员（Manager）- 24个权限

### 角色定位
租户管理员是租户内的最高权限角色，负责管理租户内的所有用户、配置租户功能、监控租户运营状况。

### 权限分类

#### 1. 用户管理（7个权限）

**作用**：管理租户内的医生账号

| 权限代码 | 权限名称 | 具体作用 | 使用场景 |
|---------|---------|---------|---------|
| `user.create_doctor` | 创建医生 | 为租户添加新的医生账号 | 新医生入职时创建账号 |
| `user.edit_doctor` | 编辑医生 | 修改医生的基本信息 | 更新医生资料、职称等 |
| `user.view_doctor` | 查看医生 | 查看医生列表和详细信息 | 查询医生信息、统计医生数量 |
| `user.enable_doctor` | 启用医生 | 激活被禁用的医生账号 | 医生返岗后重新启用账号 |
| `user.disable_doctor` | 禁用医生 | 临时禁用医生账号 | 医生离职或休假时禁用账号 |
| `user.assign_role` | 分配角色 | 为用户分配或变更角色 | 调整用户权限级别 |
| `user.reset_password` | 重置密码 | 重置用户密码 | 用户忘记密码时重置 |

**典型工作流**：
1. 新医生入职 → 使用 `create_doctor` 创建账号
2. 分配角色 → 使用 `assign_role` 设置为医生角色
3. 医生离职 → 使用 `disable_doctor` 禁用账号

#### 2. 租户配置（4个权限）

**作用**：配置租户的功能模块和运营参数

| 权限代码 | 权限名称 | 具体作用 | 使用场景 |
|---------|---------|---------|---------|
| `tenant_config.edit_basic_info` | 编辑基本信息 | 修改租户名称、地址、联系方式等 | 更新诊所信息 |
| `tenant_config.enable_modules` | 启用模块 | 开启或关闭功能模块 | 启用AI诊断、处方管理等模块 |
| `tenant_config.configure_workflow` | 配置工作流 | 设置业务流程规则 | 配置处方审核流程、病历签名流程 |
| `tenant_config.set_quotas` | 设置配额 | 设置资源使用限制 | 设置存储空间、用户数量上限 |

**典型工作流**：
1. 租户初始化 → 使用 `edit_basic_info` 设置诊所信息
2. 功能开通 → 使用 `enable_modules` 启用需要的功能
3. 流程定制 → 使用 `configure_workflow` 配置业务流程


#### 3. 租户监控（3个权限）

**作用**：监控租户的运营状况和系统使用情况

| 权限代码 | 权限名称 | 具体作用 | 使用场景 |
|---------|---------|---------|---------|
| `tenant_monitor.view_usage_stats` | 查看使用统计 | 查看用户活跃度、功能使用情况 | 分析诊所运营数据、优化资源配置 |
| `tenant_monitor.view_operation_logs` | 查看操作日志 | 查看用户操作记录 | 审计用户行为、排查问题 |
| `tenant_monitor.handle_issues` | 处理问题 | 处理系统问题和用户反馈 | 解决用户遇到的技术问题 |

**典型工作流**：
1. 日常监控 → 使用 `view_usage_stats` 查看运营数据
2. 问题排查 → 使用 `view_operation_logs` 查看操作记录
3. 问题处理 → 使用 `handle_issues` 解决问题

#### 4. 租户数据管理（3个权限）

**作用**：管理租户的数据存储和备份

| 权限代码 | 权限名称 | 具体作用 | 使用场景 |
|---------|---------|---------|---------|
| `tenant_data.view_storage_usage` | 查看存储使用 | 查看数据存储空间使用情况 | 监控存储空间、规划扩容 |
| `tenant_data.configure_backup` | 配置备份 | 设置数据备份策略 | 配置自动备份时间、频率 |
| `tenant_data.handle_data_issues` | 处理数据问题 | 处理数据相关问题 | 数据恢复、数据清理 |

#### 5. 患者管理（2个权限）

**作用**：查看租户内的患者信息（只读）

| 权限代码 | 权限名称 | 具体作用 | 使用场景 |
|---------|---------|---------|---------|
| `patient.view` | 查看患者 | 查看患者基本信息 | 统计患者数量、查询患者资料 |
| `patient.search` | 搜索患者 | 搜索患者记录 | 快速查找特定患者 |

**注意**：管理员只能查看患者信息，不能创建或编辑患者，这是为了保护患者隐私。

#### 6. 知识库（5个权限）

**作用**：查看中医知识库内容

| 权限代码 | 权限名称 | 具体作用 | 使用场景 |
|---------|---------|---------|---------|
| `knowledge.view_theory` | 查看理论 | 查看中医理论知识 | 学习中医基础理论 |
| `knowledge.view_formula` | 查看方剂 | 查看方剂信息 | 查询经典方剂 |
| `knowledge.view_herb` | 查看中药 | 查看中药信息 | 查询中药功效、用法 |
| `knowledge.view_acupoint` | 查看穴位 | 查看穴位信息 | 查询穴位位置、功效 |
| `knowledge.view_diagnosis_rules` | 查看诊断规则 | 查看AI诊断规则 | 了解AI诊断逻辑 |

### 管理员角色总结

**核心职责**：
- ✅ 管理租户内的用户账号
- ✅ 配置租户功能和流程
- ✅ 监控租户运营状况
- ✅ 管理数据存储和备份

**不能做的事**：
- ❌ 不能创建或编辑患者信息（保护隐私）
- ❌ 不能进行诊疗操作（非医疗人员）
- ❌ 不能开具处方（非医疗人员）
- ❌ 不能访问其他租户数据（多租户隔离）

## 二、中医师（Doctor）- 51个权限

### 角色定位
中医师是系统的核心用户，负责患者诊疗、开具处方、管理病历等医疗业务。

### 权限分类

#### 1. 患者管理（7个权限）

**作用**：管理医生自己的患者

| 权限代码 | 权限名称 | 具体作用 | 使用场景 |
|---------|---------|---------|---------|
| `patient.create` | 创建患者 | 为新患者建档 | 患者首次就诊时创建档案 |
| `patient.edit` | 编辑患者 | 修改患者基本信息 | 更新患者联系方式、地址等 |
| `patient.view` | 查看患者 | 查看患者详细信息 | 查看患者档案、病史 |
| `patient.search` | 搜索患者 | 搜索患者记录 | 快速查找患者 |
| `patient.advanced_search` | 高级搜索 | 按多条件搜索患者 | 按症状、诊断等条件搜索 |
| `patient.associate` | 关联患者 | 将患者关联到自己名下 | 接收转诊患者 |
| `patient.disassociate` | 解除关联 | 解除患者关联 | 患者转诊到其他医生 |

**典型工作流**：
1. 新患者就诊 → 使用 `create` 创建患者档案
2. 患者复诊 → 使用 `search` 查找患者
3. 查看病史 → 使用 `view` 查看患者详细信息
4. 接收转诊 → 使用 `associate` 关联患者


#### 2. 诊疗操作（8个权限）

**作用**：进行中医诊断和治疗

| 权限代码 | 权限名称 | 具体作用 | 使用场景 |
|---------|---------|---------|---------|
| `diagnosis.create_visit` | 创建就诊 | 为患者创建就诊记录 | 患者来诊时创建就诊单 |
| `diagnosis.edit_visit` | 编辑就诊 | 修改就诊记录 | 补充或修正就诊信息 |
| `diagnosis.view_visit` | 查看就诊 | 查看就诊记录 | 查看患者历史就诊记录 |
| `diagnosis.record_symptoms` | 记录症状 | 记录患者主诉和症状 | 记录患者描述的不适症状 |
| `diagnosis.collect_four_diagnosis` | 采集四诊 | 采集望闻问切信息 | 记录舌象、脉象等四诊信息 |
| `diagnosis.use_ai_diagnosis` | 使用AI诊断 | 使用AI辅助诊断 | 获取AI诊断建议 |
| `diagnosis.confirm_diagnosis` | 确认诊断 | 确认最终诊断结果 | 确定证型和诊断 |
| `diagnosis.modify_diagnosis` | 修改诊断 | 修改已确认的诊断 | 根据治疗效果调整诊断 |

**典型工作流**：
1. 患者就诊 → 使用 `create_visit` 创建就诊记录
2. 收集信息 → 使用 `record_symptoms` 记录症状
3. 四诊采集 → 使用 `collect_four_diagnosis` 记录四诊信息
4. AI辅助 → 使用 `use_ai_diagnosis` 获取AI建议
5. 确认诊断 → 使用 `confirm_diagnosis` 确定诊断

#### 3. 治疗方案（17个权限）

**作用**：制定和管理治疗方案

| 权限代码 | 权限名称 | 具体作用 | 使用场景 |
|---------|---------|---------|---------|
| `treatment.create` | 创建治疗方案 | 创建新的治疗方案 | 为患者制定治疗计划 |
| `treatment.edit` | 编辑治疗方案 | 修改治疗方案 | 调整治疗方案 |
| `treatment.view` | 查看治疗方案 | 查看治疗方案详情 | 查看历史治疗方案 |
| `treatment.use_ai_generation` | 使用AI生成 | 使用AI生成治疗方案 | 获取AI推荐的治疗方案 |
| `treatment.save_draft` | 保存草稿 | 保存未完成的方案 | 暂存治疗方案，稍后继续编辑 |
| `treatment.submit` | 提交方案 | 提交治疗方案 | 完成方案编辑，提交执行 |
| `treatment.view_history` | 查看历史 | 查看历史治疗方案 | 查看患者以往的治疗记录 |
| `treatment.handle_risk_warning` | 处理风险警告 | 处理配伍禁忌等警告 | 确认或调整有风险的用药 |
| `treatment.create_prescription` | 创建处方 | 开具中药处方 | 为患者开具汤剂处方 |
| `treatment.edit_prescription` | 编辑处方 | 修改处方内容 | 调整药物剂量、加减药物 |
| `treatment.create_acupuncture` | 创建针灸 | 制定针灸方案 | 为患者制定针灸治疗 |
| `treatment.create_moxibustion` | 创建艾灸 | 制定艾灸方案 | 为患者制定艾灸治疗 |
| `treatment.create_cupping` | 创建拔罐 | 制定拔罐方案 | 为患者制定拔罐治疗 |
| `treatment.create_dietary_therapy` | 创建食疗 | 制定食疗方案 | 为患者制定饮食调理方案 |
| `treatment.create_lifestyle_advice` | 创建生活建议 | 提供生活方式建议 | 给予患者起居、运动等建议 |

**典型工作流**：
1. 诊断完成 → 使用 `create` 创建治疗方案
2. AI辅助 → 使用 `use_ai_generation` 获取AI建议
3. 开具处方 → 使用 `create_prescription` 开具中药处方
4. 配伍检查 → 系统自动检查，使用 `handle_risk_warning` 处理警告
5. 其他疗法 → 使用 `create_acupuncture` 等添加其他治疗
6. 提交方案 → 使用 `submit` 提交治疗方案

#### 4. 病历管理（8个权限）

**作用**：管理患者病历文档

| 权限代码 | 权限名称 | 具体作用 | 使用场景 |
|---------|---------|---------|---------|
| `medical_record.create` | 创建病历 | 创建病历文档 | 为就诊创建病历记录 |
| `medical_record.edit` | 编辑病历 | 修改病历内容 | 补充或修正病历信息 |
| `medical_record.view` | 查看病历 | 查看病历详情 | 查看患者病历 |
| `medical_record.use_template` | 使用模板 | 使用病历模板 | 使用预设模板快速书写病历 |
| `medical_record.sign` | 签名病历 | 对病历进行电子签名 | 确认病历内容，完成签名 |
| `medical_record.print` | 打印病历 | 打印病历文档 | 为患者打印病历副本 |
| `medical_record.export` | 导出病历 | 导出病历数据 | 导出病历为PDF等格式 |
| `medical_record.share_for_consultation` | 共享会诊 | 共享病历用于会诊 | 邀请其他医生会诊 |

**典型工作流**：
1. 就诊完成 → 使用 `create` 创建病历
2. 使用模板 → 使用 `use_template` 选择病历模板
3. 编辑内容 → 使用 `edit` 填写病历内容
4. 签名确认 → 使用 `sign` 完成电子签名
5. 需要会诊 → 使用 `share_for_consultation` 共享病历

#### 5. 知识库（5个权限）

**作用**：查询中医知识库

| 权限代码 | 权限名称 | 具体作用 | 使用场景 |
|---------|---------|---------|---------|
| `knowledge.view_theory` | 查看理论 | 查看中医理论 | 学习和查询中医理论 |
| `knowledge.view_formula` | 查看方剂 | 查看方剂信息 | 查询经典方剂组成和功效 |
| `knowledge.view_herb` | 查看中药 | 查看中药信息 | 查询中药性味归经、功效 |
| `knowledge.view_acupoint` | 查看穴位 | 查看穴位信息 | 查询穴位定位和主治 |
| `knowledge.view_diagnosis_rules` | 查看诊断规则 | 查看诊断规则 | 了解辨证论治规则 |


#### 6. 医案管理（6个权限）

**作用**：管理和学习医案

| 权限代码 | 权限名称 | 具体作用 | 使用场景 |
|---------|---------|---------|---------|
| `medical_case.view_ai_generated` | 查看AI生成 | 查看AI生成的医案 | 查看系统自动生成的医案总结 |
| `medical_case.review` | 审核医案 | 审核医案内容 | 审核AI生成的医案是否准确 |
| `medical_case.edit` | 编辑医案 | 修改医案内容 | 修正或补充医案信息 |
| `medical_case.confirm` | 确认医案 | 确认医案可用 | 确认医案可以用于学习和参考 |
| `medical_case.export` | 导出医案 | 导出医案数据 | 导出医案用于学习或分享 |
| `medical_case.view_series` | 查看系列 | 查看系列医案 | 查看同一患者的系列治疗医案 |

**典型工作流**：
1. 治疗完成 → 系统自动生成医案
2. 查看医案 → 使用 `view_ai_generated` 查看AI生成的医案
3. 审核修改 → 使用 `review` 和 `edit` 审核和修改
4. 确认发布 → 使用 `confirm` 确认医案

#### 7. 会诊场景（2个权限）

**作用**：参与多医生会诊

| 权限代码 | 权限名称 | 具体作用 | 使用场景 |
|---------|---------|---------|---------|
| `consultation.view_shared_record` | 查看共享病历 | 查看其他医生共享的病历 | 查看会诊病历 |
| `consultation.participate` | 参与会诊 | 参与会诊讨论 | 提供会诊意见 |

**典型工作流**：
1. 收到会诊邀请 → 使用 `view_shared_record` 查看病历
2. 提供意见 → 使用 `participate` 参与会诊讨论

### 医生角色总结

**核心职责**：
- ✅ 管理自己的患者
- ✅ 进行诊断和治疗
- ✅ 开具处方和制定治疗方案
- ✅ 管理病历文档
- ✅ 参与会诊

**不能做的事**：
- ❌ 不能管理其他医生的账号
- ❌ 不能配置租户设置
- ❌ 不能访问其他医生的患者（除非会诊）
- ❌ 不能访问其他租户的数据

## 三、患者（Patient）- 11个权限

### 角色定位
患者是系统的服务对象，可以查看自己的诊疗信息和病历，但不能修改医疗数据。

### 权限分类

#### 1. 个人信息（2个权限）

**作用**：查看个人档案信息

| 权限代码 | 权限名称 | 具体作用 | 使用场景 |
|---------|---------|---------|---------|
| `personal.view_basic` | 查看基本信息 | 查看姓名、年龄、联系方式等 | 查看和确认个人基本信息 |
| `personal.view_medical` | 查看医疗信息 | 查看过敏史、既往病史等 | 查看个人医疗档案 |

#### 2. 诊疗记录查看（3个权限）

**作用**：查看自己的诊疗记录

| 权限代码 | 权限名称 | 具体作用 | 使用场景 |
|---------|---------|---------|---------|
| `diagnosis_view.view_history` | 查看历史 | 查看历史就诊记录 | 查看以往的就诊时间和诊断 |
| `diagnosis_view.view_four_diagnosis` | 查看四诊 | 查看四诊信息 | 查看医生记录的症状和体征 |
| `diagnosis_view.view_diagnosis_result` | 查看诊断结果 | 查看诊断结论 | 查看医生的诊断和证型 |

#### 3. 治疗方案查看（3个权限）

**作用**：查看治疗方案和处方

| 权限代码 | 权限名称 | 具体作用 | 使用场景 |
|---------|---------|---------|---------|
| `treatment_view.view_prescription` | 查看处方 | 查看中药处方 | 查看处方内容、用法用量 |
| `treatment_view.view_non_drug_therapy` | 查看非药物疗法 | 查看针灸、艾灸等方案 | 查看非药物治疗方案 |
| `treatment_view.view_advice` | 查看建议 | 查看医生的医嘱和建议 | 查看饮食、起居等建议 |

#### 4. 病历查看（3个权限）

**作用**：查看和导出病历

| 权限代码 | 权限名称 | 具体作用 | 使用场景 |
|---------|---------|---------|---------|
| `medical_record_view.view` | 查看病历 | 查看病历内容 | 查看完整的病历记录 |
| `medical_record_view.print` | 打印病历 | 打印病历文档 | 打印病历用于报销或存档 |
| `medical_record_view.export` | 导出病历 | 导出病历数据 | 导出病历用于其他医院就诊 |

### 患者角色总结

**核心职责**：
- ✅ 查看个人信息
- ✅ 查看诊疗记录
- ✅ 查看治疗方案和处方
- ✅ 查看和导出病历

**不能做的事**：
- ❌ 不能修改任何医疗数据
- ❌ 不能查看其他患者的信息
- ❌ 不能进行诊疗操作
- ❌ 不能开具处方

**设计理念**：
- 患者只有查看权限，没有修改权限
- 保护医疗数据的完整性和准确性
- 患者可以导出自己的病历用于其他用途

## 四、药剂师（Pharmacist）- 6个权限

### 角色定位
药剂师负责审核处方、配药发药，需要查看患者基本信息和处方内容。

### 权限分类

#### 1. 患者基本信息查看（2个权限）

**作用**：查看患者基本信息以确认身份

| 权限代码 | 权限名称 | 具体作用 | 使用场景 |
|---------|---------|---------|---------|
| `patient.view` | 查看患者 | 查看患者基本信息 | 确认患者身份 |
| `patient.search` | 搜索患者 | 搜索患者记录 | 查找患者处方 |

#### 2. 处方查看（2个权限）

**作用**：查看和审核处方

| 权限代码 | 权限名称 | 具体作用 | 使用场景 |
|---------|---------|---------|---------|
| `treatment.view` | 查看治疗方案 | 查看处方内容 | 审核处方、配药 |
| `treatment.view_history` | 查看历史 | 查看历史处方 | 查看患者用药历史 |

### 药剂师角色总结

**核心职责**：
- ✅ 查看患者基本信息
- ✅ 查看和审核处方
- ✅ 查询中药和方剂信息

**不能做的事**：

**设计理念**：
- 药剂师只能查看与配药相关的信息

- 可以查询中药信息以确保配药准确

## 六、权限设计原则

### 1. 最小权限原则

每个角色只拥有完成其工作所必需的最小权限集。

**示例**：
- 患者只能查看，不能修改医疗数据
- 药剂师只能查看处方，不能查看完整病历
- 管理员不能进行诊疗操作

### 2. 职责分离原则

不同角色的职责明确分离，避免权限冲突。

**示例**：
- 医生负责诊疗，管理员负责管理
- 患者只能查看自己的数据
- 药剂师只能查看与配药相关的信息

### 3. 数据隔离原则

确保租户间、用户间的数据隔离。

**示例**：
- 医生只能访问自己的患者
- 租户只能访问自己的数据
- 跨租户访问需要特殊授权

### 4. 审计追踪原则

所有权限验证和敏感操作都会被记录。

**记录内容**：
- 谁（用户）
- 什么时候（时间）
- 做了什么（操作）
- 在哪里（IP地址）
- 结果如何（成功/失败）

### 5. 临时权限原则

特殊场景使用临时权限，自动过期。

**特点**：
- 有明确的有效期
- 需要审批流程
- 自动撤销
- 详细记录使用情况

## 八、权限验证流程

### 1. API请求权限验证

```
用户请求 → 身份认证 → 权限验证 → 业务逻辑 → 返回结果
```

**详细步骤**：
1. 用户发起API请求
2. 系统验证用户身份（JWT Token）
3. 提取用户ID和租户ID
4. 查询用户权限（从缓存或数据库）
5. 验证是否具有所需权限
6. 如果有权限，执行业务逻辑
7. 如果无权限，返回403错误
8. 记录审计日志

### 2. 权限缓存机制

**缓存策略**：
- 用户权限缓存30分钟
- 角色权限缓存60分钟
- 权限变更时自动刷新缓存

**缓存键格式**：
- 用户权限：`permissions:user:{userId}:tenant:{tenantId}`
- 角色权限：`permissions:role:{roleId}`

### 3. 多租户隔离

**隔离机制**：
- 每个请求都会验证租户ID
- 数据查询自动添加租户过滤条件
- 跨租户访问需要特殊权限

**验证流程**：
```
请求 → 提取租户ID → 验证权限 → 添加租户过滤 → 查询数据
```

## 九、常见问题

### Q1：为什么管理员不能创建患者？

**答**：这是基于职责分离和隐私保护的设计：
- 管理员是管理角色，不是医疗角色
- 患者信息应该由医生创建和管理
- 避免管理员滥用权限访问患者隐私

### Q2：为什么药剂师不能查看完整病历？

**答**：这是基于最小权限原则：
- 药剂师只需要处方信息来配药
- 完整病历包含患者隐私信息
- 限制访问范围可以更好地保护隐私

### Q3：临时权限如何管理？

**答**：临时权限有严格的管理机制：
- 必须经过审批流程
- 有明确的有效期
- 自动过期撤销
- 所有使用都会被记录

### Q4：如何处理权限冲突？

**答**：系统采用明确的权限优先级：
1. 拒绝优先于允许
2. 临时权限优先于永久权限
3. 特定权限优先于通用权限

### Q5：权限变更后多久生效？

**答**：
- 立即生效：新的请求会使用新权限
- 缓存刷新：权限变更会自动刷新相关缓存
- 已登录用户：下次请求时会使用新权限
