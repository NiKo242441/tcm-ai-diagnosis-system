## 🎯 核心功能特性

### 1. 权限验证特性

```csharp
// 单个权限
[RequirePermission(PermissionConstants.KnowledgeBase.ViewHerb)]

// 多个权限（需要全部）
[RequirePermission(
    PermissionConstants.PatientManagement.Edit,
    PermissionConstants.PatientManagement.View
)]

// 多个权限（需要任一）
[RequirePermission(
    PermissionConstants.PatientManagement.View,
    PermissionConstants.PatientManagement.Search,
    Mode = PermissionCheckMode.RequireAny
)]
```

### 2. 权限验证流程

```
HTTP请求
  ↓
白名单检查 → 是 → 放行
  ↓ 否
提取权限要求 → 无要求 → 放行
  ↓ 有要求
用户认证检查 → 未认证 → 返回401
  ↓ 已认证
权限验证 → 无权限 → 记录日志 → 返回403
  ↓ 有权限
放行请求
```

### 3. 缓存机制

- **用户权限缓存**：30分钟过期（可配置）
- **角色权限缓存**：60分钟过期
- **缓存键格式**：
  - 用户权限：`user_permissions:{userId}:{tenantId}`
  - 角色权限：`role_permissions:{roleId}`

### 4. 审计日志

所有权限验证失败的尝试都会记录到 `permission_change_logs` 表：
- 用户ID
- 租户ID
- 请求路径
- 所需权限
- IP地址
- 时间戳

### 5. 多租户支持

- 权限检查时自动验证租户ID
- 支持跨租户临时权限（转诊场景）
- 租户隔离确保数据安全

## 🔧 配置说明

### appsettings.json 配置

```json
{
  "PermissionSettings": {
    "EnablePermissionCheck": true,           // 是否启用权限检查
    "CacheExpirationMinutes": 30,           // 缓存过期时间（分钟）
    "WhitelistPaths": [                     // 白名单路径
      "/api/auth/login",
      "/api/health",
      "/swagger",
      "/Identity/Account/Login",
      "/Identity/Account/Logout",
      "/Identity/Account/Register",
      "/Identity/Account/AccessDenied",
      "/css",
      "/js",
      "/lib",
      "/favicon.ico"
    ],
    "AuditLogLevel": "Warning",             // 审计日志级别
    "EnableDetailedErrors": false           // 是否启用详细错误（仅开发环境）
  }
}
```

### 快速启用/禁用权限检查

```json
{
  "PermissionSettings": {
    "EnablePermissionCheck": false  // 设置为false可立即禁用权限检查
  }
}
```

## 🚀 使用指南

### 1. 为控制器添加权限标注

```csharp
using TcmAiDiagnosis.Web.Attributes;
using TcmAiDiagnosis.Domain.Constants;

[ApiController]
[Route("api/[controller]")]
public class PatientController : ControllerBase
{
    [HttpGet]
    [RequirePermission(PermissionConstants.PatientManagement.View)]
    public async Task<IActionResult> GetPatients()
    {
        // 方法实现
    }

    [HttpPost]
    [RequirePermission(PermissionConstants.PatientManagement.Create)]
    public async Task<IActionResult> CreatePatient([FromBody] PatientDto dto)
    {
        // 方法实现
    }
}
```

### 2. 刷新用户权限缓存

当用户的角色或权限发生变更时，需要刷新缓存：

```csharp
public class UserManagementService
{
    private readonly IPermissionService _permissionService;

    public async Task UpdateUserRoles(int userId, int? tenantId)
    {
        // 更新用户角色...
        
        // 刷新权限缓存
        await _permissionService.RefreshUserPermissionCacheAsync(userId, tenantId);
    }
}
```

### 3. 在业务逻辑中检查权限

```csharp
public class PatientService
{
    private readonly IPermissionService _permissionService;

    public async Task<bool> CanAccessPatient(int userId, int? tenantId, int patientId)
    {
        // 检查用户是否有查看患者的权限
        return await _permissionService.CheckPermissionAsync(
            userId,
            tenantId,
            PermissionConstants.PatientManagement.View
        );
    }
}
```

## 📈 性能指标

### 预期性能

- **首次权限检查**：~50ms（数据库查询）
- **缓存命中**：<1ms（内存读取）
- **缓存命中率**：预计 >95%
- **对整体响应时间影响**：<5%

### 优化建议

1. **数据库索引**（可选）：
```sql
-- 优化权限查询性能
CREATE INDEX idx_role_permissions_lookup 
ON role_permissions(role_id, permission_id, is_granted);

CREATE INDEX idx_permissions_code 
ON permissions(permission_code, is_active);

CREATE INDEX idx_temp_permissions_user_status 
ON temporary_permissions(user_id, status, valid_from, valid_to);
```

2. **缓存策略调整**：
   - 根据实际使用情况调整缓存过期时间
   - 高频访问的权限可以延长缓存时间

## ⚠️ 注意事项

### 1. 白名单路径

确保以下路径在白名单中，避免影响正常功能：
- 登录/登出路径
- 健康检查路径
- Swagger文档路径
- 静态资源路径（CSS、JS、图片等）

### 2. 用户声明配置

权限中间件依赖以下用户声明：
- `ClaimTypes.NameIdentifier`：用户ID
- `TenantId`：租户ID（可选）

确保在用户登录时正确设置这些声明。

### 3. 错误响应格式

权限验证失败时返回标准JSON格式：

**401 Unauthorized**：
```json
{
  "error": "Unauthorized",
  "message": "用户未认证，请先登录",
  "timestamp": "2026-03-03T10:30:00Z",
  "path": "/api/patients"
}
```

**403 Forbidden**：
```json
{
  "error": "Forbidden",
  "message": "权限不足，无法访问此资源",
  "requiredPermissions": ["patient.view"],
  "timestamp": "2026-03-03T10:30:00Z",
  "path": "/api/patients"
}
```
